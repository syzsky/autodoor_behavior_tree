## 1. 产品概述

### 1.1 什么是行为树？

行为树是一种可视化的自动化流程设计工具，通过图形化界面拖拽节点并连接它们，可以创建复杂的自动化任务。行为树广泛应用于客户端自动化、RPA自动化、测试自动化等领域。

### 1.2 产品定位

AutoDoor 行为树系统是一个独立的可视化行为树编辑与执行框架，面向 Windows 平台的自动化场景

### 1.3 核心特性

| 特性       | 说明                                     |
| -------- | -------------------------------------- |
| 可视化编辑器   | 基于 CustomTkinter 的节点式编辑器，支持拖拽、连线、缩放、框选 |
| 行为树引擎    | 独立线程执行，支持启动/暂停/停止/恢复                   |
| 20 种内置节点 | 4 种复合节点 + 6 种条件节点 + 10 种动作节点           |
| 黑板系统     | 观察者模式的数据共享机制，节点间解耦通信                   |
| 撤销/重做    | Command 模式，支持 100 步历史                  |
| DD 虚拟键盘  | 硬件模拟输入，绕过大多数输入检测                       |
| OCR 集成   | 内嵌 RapidOCR，支持中英文识别                    |
| 脚本录制     | TXT 脚本录制与回放                            |
| 项目管理     | 完整的项目文件管理系统，支持导入导出                     |

### 1.4 版本说明

系统提供两种打包版本：

| 版本  | 输入方式      | 适用场景           |
| --- | --------- | -------------- |
| 标准版 | PyAutoGUI | 普通应用自动化        |
| DD版 | DD64.dll  | 客户端自动化（绕过输入检测） |

## 2. 快速入门

### 2.1 创建第一个行为树

**步骤1：创建新文件**

- 点击工具栏的【新建】按钮
- 或按快捷键 `Ctrl + N`

**步骤2：添加节点**

- 从左侧节点面板点击需要的节点
- 节点会自动添加到画布中央

**步骤3：连接节点**

- 从父节点底部的输出端口拖拽到子节点顶部的输入端口
- 释放鼠标完成连接

**步骤4：配置属性**

- 点击选中节点
- 在右侧属性面板配置参数

**步骤5：保存运行**

- 点击【保存】按钮保存文件
- 点击【运行】按钮执行行为树

```
│   结果取反: [ ]    │
│   失败重试: [0]    │
│   超时时间: [0]    │
└────────────────────┘
```

## 4. 节点类型详解

### 4.1 开始节点

**用途**：行为树的根节点，作为整个行为树的入口点，控制整体执行流程。

**特性**：

| 特性   | 说明                   |
| ---- | -------------------- |
| 自动创建 | 新建项目时自动创建，无需手动添加     |
| 不可删除 | 开始节点受保护，无法被删除、复制或剪切  |
| 失败继续 | 子节点失败后继续执行后续子节点（不短路） |
| 循环控制 | 支持重复执行次数控制           |

**执行流程**：

```
        开始
          │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3 ...
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
    重复/完成
```

**参数**：

| 参数     | 说明                            |
| ------ | ----------------------------- |
| 失败重试次数 | 子节点失败后的重试次数（-1 表示无限重试，默认 0）   |
| 重复次数   | 整个行为树的重复执行次数（-1 表示无限循环，默认 -1） |
| 重复间隔   | 重复执行之间的间隔时间（毫秒，默认 100）        |
| 超时时间   | 超时时间（毫秒，0 表示不限时）              |
| 绑定窗口   | 是否启用窗口绑定功能（默认关闭）              |
| 窗口标题   | 目标窗口标题关键字（启用绑定窗口后显示）          |

**窗口绑定功能**：

启用窗口绑定后，行为树将绑定到指定窗口，实现以下功能：

| 功能     | 说明                                 |
| ------ | ---------------------------------- |
| 后台截图   | 使用 PrintWindow API 截取后台窗口图像，无需窗口置顶 |
| 自动坐标转换 | 屏幕绝对坐标与窗口相对坐标自动转换                  |
| 智能窗口切换 | 执行动作前自动切换到绑定窗口，执行后恢复原窗口            |

**窗口绑定使用步骤**：

1. 点击开始节点，打开属性面板
2. 勾选"绑定窗口"选项
3. 在"窗口标题"下拉框中选择目标窗口
   - 下拉列表显示所有可见窗口的标题
   - 支持关键字模糊匹配
   - 点击下拉框自动刷新窗口列表
   - 点击"清空"按钮可取消窗口绑定
4. 配置其他参数后运行行为树

**窗口绑定注意事项**：

- 目标窗口必须处于可见状态（未被最小化）
- 区域选择时会自动转换为窗口相对坐标
- 执行动作时会自动切换到绑定窗口，执行完成后恢复原窗口
- 如果目标窗口未找到，行为树会继续执行但不会绑定窗口
- 延时、设变量、报警、代码、脚本节点不会触发窗口切换

**重复次数说明**：

| 次数    | 说明                      |
| ----- | ----------------------- |
| -1    | 无限循环（默认），行为树会一直执行直到手动停止 |
| 0     | 不重复，执行一次后结束             |
| N > 0 | 执行 N 次后结束               |

**使用场景**：

- 作为行为树的入口点
- 控制整体执行流程和重复次数
- 实现持续运行的自动化任务

**示例**：

```
开始节点（重复次数=-1）
├── OCR检测：检测目标
├── 鼠标点击：点击目标
└── 延时：等待1秒
```

**注意事项**：

- 每个行为树有且仅有一个开始节点
- 开始节点会自动创建在画布中央偏上的位置
- 无法通过节点面板手动添加开始节点

***

### 4.2 组合节点

组合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 4.2.1 顺序节点

**用途**：按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

**执行流程**：

```
          顺序
           │
   ┌───────┼────────┐
   ↓       ↓        ↓
子节点1 → 子节点2 → 子节点3 → ...
  ↓失败     ↓失败     ↓失败
返回失败   返回失败   返回失败
```

**参数**：

| 参数      | 说明                  |
| ------- | ------------------- |
| 子节点间隔   | 每个子节点执行完成后的等待时间（毫秒） |
| 失败后继续执行 | 开启后，中间节点失败仍继续执行后续节点 |

**使用场景**：

- 需要按固定顺序执行的任务
- 流程化的自动化操作
- 登录流程、表单填写等

**示例**：

```
顺序节点
├── OCR检测：检测登录按钮
├── 鼠标点击：点击登录按钮
├── 延时：等待2秒
└── OCR检测：检测登录成功
```

#### 4.2.2 选择节点

**用途**：按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

**执行流程**：

```
子节点1 → 子节点2 → 子节点3 → ...
  ↓成功     ↓成功     ↓成功
返回成功   返回成功   返回成功
```

**参数**：

| 参数    | 说明                  |
| ----- | ------------------- |
| 子节点间隔 | 每个子节点执行完成后的等待时间（毫秒） |

**使用场景**：

- 多种方案尝试，任一成功即可
- 容错处理，提供备选方案

**示例**：

```
选择节点
├── OCR检测：检测账号密码登录入口
├── OCR检测：检测扫码登录入口
└── OCR检测：检测第三方登录入口
```

#### 4.2.3 并行节点

**用途**：同时执行所有子节点，根据成功策略决定返回条件。

**执行流程**：

```
       并行
         │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
     汇总结果
```

**参数**：

| 参数   | 说明                              |
| ---- | ------------------------------- |
| 成功策略 | 全部成功：所有子节点成功才成功；任一成功：任一子节点成功即成功 |

**使用场景**：

- 同时监控多个条件
- 并行执行独立任务

#### 4.2.4 随机节点

**用途**：随机执行子节点，根据成功策略决定返回条件。

**执行流程**：

```
         随机
           │
   ┌───────┼───────┐
   ↓       ↓       ↓
子节点?  子节点?  子节点?
   │       │       │
   └───────┼───────┘
           ↓
       汇总结果
```

**参数**：

| 参数      | 说明                               |
| ------- | -------------------------------- |
| 成功策略    | 全部成功：所有子节点成功才成功；任一成功：任一子节点成功即成功  |
| 失败后继续执行 | 开启后，子节点失败不会中断随机执行（仅"全部成功"策略生效）   |
| 完全随机    | 开启后，已执行的子节点仍可被再次选中，直到所有子节点都执行过一次 |

**执行逻辑说明**：

| 设置组合             | 执行行为                    |
| ---------------- | ----------------------- |
| 全部成功 + 失败后继续执行关闭 | 遇到失败立即返回失败              |
| 全部成功 + 失败后继续执行开启 | 遇到失败继续执行，所有子节点执行完毕后判断   |
| 任一成功             | 遇到成功立即返回成功              |
| 完全随机关闭           | 每个子节点只执行一次，执行后从随机池移除    |
| 完全随机开启           | 子节点可被重复选中，直到所有子节点都执行过一次 |

**使用场景**：

- 随机选择不同的执行路径
- 模拟随机行为
- 多种方案随机尝试

**示例**：

```
随机节点（成功策略=任一成功）
├── OCR检测：检测入口A
├── OCR检测：检测入口B
└── OCR检测：检测入口C

// 随机选择一个入口检测，成功即返回
```

### 4.3 条件节点

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

#### 4.3.1 文字检测节点

**用途**：检测屏幕指定区域的文字内容。

**参数**：

| 参数    | 说明                         |
| ----- | -------------------------- |
| 检测区域  | 屏幕上的检测范围                   |
| 关键词   | 要检测的文字（多个用逗号分隔）            |
| 语言    | OCR识别语言（简体中文/English/繁体中文） |
| 图像预处理 | 预处理模式（默认/复杂色彩）             |
| 位置变量名 | 保存检测位置的黑板变量名               |
| 坐标偏移  | 检测位置的偏移量                   |

**语言支持**：

| 语言选项    | 说明         |
| ------- | ---------- |
| 简体中文    | 简体中文字符（默认） |
| English | 英文字符       |
| 繁体中文    | 繁体中文字符     |

**使用技巧**：

- 关键词可以是多个，用逗号分隔
- 检测成功后会自动保存位置到黑板，供点击节点使用
- 中文识别已针对中文文字特点进行优化
- 默认语言为简体中文，适合大多数中文场景

#### 4.3.2 图像匹配节点

**用途**：检测屏幕指定区域是否包含目标图像。

**参数**：

| 参数    | 说明                  |
| ----- | ------------------- |
| 检测区域  | 屏幕上的检测范围            |
| 模板路径  | 目标图像文件路径            |
| 匹配阈值  | 匹配度百分比（0\~100），默认80 |
| 位置变量名 | 保存检测位置的黑板变量名        |

**获取模板图像**：

1. 点击【截图】按钮
2. 工具窗口自动最小化
3. 在屏幕上拖拽选择要截取的区域
4. 截图自动保存到项目目录

**使用技巧**：

- 阈值越高，匹配越精确
- 建议阈值设置在 80\~90 之间
- 模板图像应尽量清晰、特征明显

#### 4.3.3 颜色检测节点

**用途**：检测屏幕指定区域是否包含目标颜色。

**参数**：

| 参数    | 说明                |
| ----- | ----------------- |
| 检测区域  | 屏幕上的检测范围          |
| 目标颜色  | 要检测的颜色（RGB值）      |
| 容差    | 颜色允许的偏差范围（0\~100） |
| 最小像素数 | 最少匹配像素数           |
| 位置变量名 | 保存检测位置的黑板变量名      |

**使用技巧**：

- 容差越大，匹配越宽松
- 建议容差设置在 10\~30 之间

#### 4.3.4 数字比较节点

**用途**：识别屏幕上的数字并进行比较判断。

**参数**：

| 参数    | 说明                     |
| ----- | ---------------------- |
| 检测区域  | 屏幕上的检测范围               |
| 语言    | OCR识别语言                |
| 比较运算符 | 比较运算符（>、>=、<、<=、==、!=） |
| 比较值   | 用于比较的阈值                |
| 位置变量名 | 保存检测位置的黑板变量名           |

**使用技巧**：

- 识别成功后数字值保存到黑板变量 `last_number_value`
- 可配合变量判断节点实现复杂逻辑

#### 4.3.5 变量判断节点

**用途**：判断黑板变量是否满足指定条件。

**参数**：

| 参数  | 说明        |
| --- | --------- |
| 变量名 | 要判断的黑板变量名 |
| 运算符 | 比较运算符     |
| 比较值 | 用于比较的值    |

**支持的运算符**：

| 运算符           | 说明          |
| ------------- | ----------- |
| ==            | 等于          |
| !=            | 不等于         |
| >             | 大于          |
| <             | 小于          |
| >=            | 大于等于        |
| <=            | 小于等于        |
| exists        | 变量存在        |
| not\_exists   | 变量不存在       |
| contains      | 包含（字符串/列表）  |
| not\_contains | 不包含（字符串/列表） |

#### 4.3.6 文本提取节点

**用途**：通过 OCR 识别屏幕指定区域的文字内容，提取文本并保存到黑板变量，供文本输入节点使用。

**参数**：

| 参数      | 说明                                     |
| ------- | -------------------------------------- |
| 检测区域    | 屏幕上的检测范围                               |
| 提取模式    | 提取方式（all/keywords）                     |
| 关键词     | 要匹配的关键词（keywords模式时显示）                 |
| 语言      | OCR识别语言（简体中文/English/繁体中文）             |
| 图像预处理   | 预处理模式（默认/复杂色彩）                         |
| 输出变量名   | 提取文本保存的黑板变量名（默认 last\_extracted\_text） |
| 保存全部文本  | 是否保存完整OCR识别结果                          |
| 全部文本变量名 | 完整文本保存的变量名（开启保存全部文本时显示）                |
| 保存位置    | 是否保存检测区域中心坐标                           |
| 位置变量名   | 位置保存的变量名                               |
| 坐标偏移    | 检测位置的偏移量                               |

**提取模式说明**：

| 模式       | 说明           |
| -------- | ------------ |
| all      | 提取区域内的全部文本   |
| keywords | 仅提取包含关键词的文本行 |

**使用技巧**：

- 文本提取节点与文本输入节点配合使用，可实现"识别文字→输入文字"的自动化流程
- 默认输出变量名为 `last_extracted_text`，与文本输入节点的默认黑板变量名一致，无需手动配置即可配合使用
- 如需多个文本提取节点，可通过自定义输出变量名区分
- 开启"保存全部文本"可获取完整的 OCR 识别结果，用于调试或复杂文本处理

**典型使用场景**：

```
文本提取节点（提取验证码）
    ↓
延时节点（等待1秒）
    ↓
文本输入节点（输入验证码）
```

***

### 4.4 动作节点

动作节点执行具体操作，产生副作用。

#### 4.4.1 按键节点

**用途**：模拟键盘按键操作。

**参数**：

| 参数   | 说明                    |
| ---- | --------------------- |
| 按键   | 要按下的键                 |
| 动作   | 按键动作类型（press/down/up） |
| 按住时长 | 按住按键的时间（毫秒）           |

**动作类型**：

| 类型    | 说明             |
| ----- | -------------- |
| press | 完整点击（按下→等待→抬起） |
| down  | 仅按下按键          |
| up    | 仅抬起按键          |

**常用按键**：

- 字母：a\~z
- 数字：0\~9
- 功能键：f1\~f12
- 控制键：ctrl, alt, shift, win
- 方向键：up, down, left, right
- 其他：enter, space, tab, escape

#### 4.4.2 鼠标点击节点

**用途**：模拟鼠标点击操作，支持单击、双击、多击和无限点击。

**参数**：

| 参数      | 说明                    |
| ------- | --------------------- |
| 按钮      | 鼠标按钮（左键/右键/中键）        |
| 动作      | 点击动作类型（press/down/up） |
| 按住时长    | 按住鼠标的时间（毫秒）           |
| 位置      | 点击位置（可选）              |
| 点击最近检测点 | 勾选后使用最近检测到的位置         |
| 位置变量名   | 检测位置的黑板变量名            |
| 点击次数    | 点击次数（1-10次，-1为无限点击）   |
| 点击间隔    | 多次点击之间的间隔时间（毫秒）       |

**点击次数说明**：

| 次数   | 说明           |
| ---- | ------------ |
| 1    | 单击（默认）       |
| 2    | 双击           |
| 3\~N | 多击           |
| -1   | 无限点击（直到手动停止） |

#### 4.4.3 鼠标移动节点

**用途**：移动鼠标位置，支持普通移动、相对移动和拖拽操作。

**参数**：

| 参数          | 说明                     |
| ----------- | ---------------------- |
| 起点位置(相对移动值) | 目标位置（相对移动时为偏移值）        |
| 移动到最近检测点    | 勾选后使用最近检测到的位置          |
| 黑板变量名       | 黑板变量名                  |
| 相对移动        | 是否相对当前位置移动（开启后隐藏拖拽参数）  |
| 平滑移动        | 是否启用平滑移动效果             |
| 移动类型        | 移动或拖拽                  |
| 拖拽按键        | 拖拽时按住的鼠标按钮（相对移动时隐藏）    |
| 终点          | 拖拽的目标位置（相对移动时隐藏）       |
| 终点使用黑板位置    | 使用黑板变量作为终点位置（相对移动时隐藏）  |
| 终点位置变量名     | 终点位置的黑板变量名（相对移动时隐藏）    |
| 拖拽时长        | 拖拽过程时长（毫秒，相对移动时隐藏）     |
| 拖拽时长随机范围    | 拖拽时长的随机范围（±毫秒，相对移动时隐藏） |

**移动类型**：

| 类型 | 说明             |
| -- | -------------- |
| 移动 | 普通移动，直接移动到目标位置 |
| 拖拽 | 按住鼠标按钮并移动到目标位置 |

**相对移动说明**：

| 设置     | 行为          |
| ------ | ----------- |
| 相对移动关闭 | 移动到屏幕绝对坐标位置 |
| 相对移动开启 | 从当前位置偏移指定距离 |

**使用场景**：

- **普通移动**：移动到固定位置或检测到的位置
- **相对移动**：从当前位置偏移一小段距离（如点击按钮后移动到旁边）
- **拖拽操作**：将对象从一个位置拖到另一个位置

#### 4.4.4 鼠标滚轮节点

**用途**：模拟鼠标滚轮滚动操作。

**参数**：

| 参数   | 说明          |
| ---- | ----------- |
| 滚动距离 | 每次滚动的距离     |
| 滚动次数 | 滚动次数        |
| 滚动方向 | 向上/向下/向左/向右 |

#### 4.4.5 延时节点

**用途**：等待指定时间。

**参数**：

| 参数   | 说明       |
| ---- | -------- |
| 延时时长 | 等待时间（毫秒） |

**使用场景**：

#### 4.4.10 文本输入节点

**用途**：从黑板变量、预设文本或文件获取文本内容，逐字符输入到当前焦点窗口。

**参数**：

| 参数       | 说明                         |
| -------- | -------------------------- |
| 输入模式     | 文本来源（文本提取值/预设文本/文件）        |
| 预设文本     | 预设的文本列表，每行一条（预设文本模式时显示）    |
| 执行模式     | 预设文本的选择方式（顺序/随机，预设文本模式时显示） |
| 黑板变量名    | 获取文本的黑板变量名（文本提取值模式时显示）     |
| 文件路径     | 文本文件路径（文件模式时显示）            |
| 输入间隔(ms) | 每个字符输入的间隔时间（毫秒）            |
| 输入前清空    | 输入前是否清空已有内容（Ctrl+A全选后覆盖）   |
| 保存输入文本   | 是否将输入的文本保存到黑板变量            |
| 输出变量名    | 保存输入文本的变量名（开启保存输入文本时显示）    |

**输入模式说明**：

| 模式    | 说明        | 适用场景               |
| ----- | --------- | ------------------ |
| 文本提取值 | 从黑板变量获取文本 | 与文本提取节点配合，输入识别到的文字 |
| 预设文本  | 从预设文本列表获取 | 输入固定的账号、密码等        |

**执行模式说明**（预设文本模式）：

| 模式 | 说明                           |
| -- | ---------------------------- |
| 顺序 | 按顺序循环使用预设文本（第一条→第二条→...→第一条） |
| 随机 | 每次随机选择一条预设文本                 |

**使用技巧**：

- 文本输入采用逐字符 Ctrl+V 粘贴方式，兼容各种输入框
- 输入间隔建议设置 30-100ms，过快可能导致字符丢失
- 开启"输入前清空"可替换输入框中的已有内容
- 开启"保存输入文本"可将输入内容保存到黑板，供后续节点判断使用
- 文件模式支持 .txt 文本文件，文件会自动保存到项目文件夹中

**典型使用场景**：

**场景1：识别验证码并输入**

```

    ↓
延时节点（等待0.5秒）
    ↓
鼠标点击节点（点击验证码输入框）
    ↓
延时节点（等待0.3秒）
    ↓

```

**场景2：输入预设账号**

```
鼠标点击节点（点击账号输入框）
    ↓
文本输入节点（输入模式=预设文本，预设文本=账号列表）
```

**场景3：从文件输入长文本**

```
鼠标点击节点（点击文本输入区域）
    ↓
文本输入节点（输入模式=文件，文件路径=文本文件）
```

## 6. 连接操作

### 6.1 节点端口说明

每个节点有两个端口：

- **输入端口**：节点顶部的圆点，用于接收父节点的连接
- **输出端口**：节点底部的圆点，用于连接子节点

```
        ○ 输入端口
    ┌─────────┐
    │  节点    │
    └─────────┘
        ○ 输出端口
```

### 6.2 创建连接

1. 将鼠标移到父节点的**输出端口**（底部圆点）
2. 光标变为十字形状时，按住左键拖拽
3. 拖拽到子节点的**输入端口**（顶部圆点）
4. 释放鼠标完成连接

### 6.3 删除连接

| 方法  | 操作              |
| --- | --------------- |
| 快捷键 | 选中连线后按 `Delete` |

### 6.4 连接规则

- 一个输出端口可以连接多个子节点
- 一个输入端口只能连接一个父节点
- 不能创建循环连接（子节点连接回父节点）

### 6.5 执行顺序

当组合节点有多个子节点时，连接线上会显示序号（1、2、3...）表示执行顺序。

3. 文件路径自动填入

### 8.6 坐标偏移测量

条件节点（OCR检测、图像匹配、颜色检测、数字比较）支持坐标偏移功能，可以将检测到的位置偏移后再保存到黑板：

**使用偏移测量工具**：

1. 在条件节点的属性面板中找到【坐标偏移】字段
2. 点击【测量】按钮
3. **第一步**：点击参考点（检测到的原始位置）
4. **第二步**：点击目标点（实际需要点击的位置）
5. 偏移值自动计算并填入

**偏移测量流程**：

```
┌─────────────────────────────────────────────────────────────┐
│                     偏移测量流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 点击【测量】按钮                                         │
│       ↓                                                     │
│  2. 放大镜显示，点击参考点（如按钮中心）                      │
│       ↓                                                     │
│  3. 放大镜继续显示，点击目标点（如按钮边缘）                  │
│       ↓                                                     │
│  4. 自动计算偏移值：(目标X-参考X, 目标Y-参考Y)               │
│       ↓                                                     │
│  5. 偏移值填入输入框                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**使用场景**：

- 检测到按钮位置后，需要点击按钮的特定边缘
- OCR识别文字后，需要点击文字旁边的图标
- 图像匹配后，需要点击匹配图像的相对位置

### 8.7 随机范围设置

为了模拟更自然的人类操作，部分节点支持为时长参数设置随机范围：

**支持的节点和参数**：

| 节点     | 参数   | 随机范围说明            |
| ------ | ---- | ----------------- |
| 延时节点   | 延时时长 | 实际延时 = 基础值 ± 随机范围 |
| 鼠标点击节点 | 点击间隔 | 实际间隔 = 基础值 ± 随机范围 |
| 鼠标移动节点 | 移动时长 | 实际时长 = 基础值 ± 随机范围 |

**设置随机范围**：

1. 在节点属性面板中找到时长参数
2. 在参数右侧找到【随机范围】输入框
3. 输入随机范围值（毫秒）
4. 实际执行时会在 \[基础值-范围, 基础值+范围] 之间随机选择

**示例**：

```
延时时长: 1000ms
随机范围: 200ms

实际延时范围: 800ms ~ 1200ms
每次执行随机选择一个值
```

**使用建议**：

- 随机范围建议设置为时长的 10%\~20%
- 适当的随机变化可以模拟更自然的人类操作
- 避免设置过大的随机范围影响业务逻辑

### 8.8 装饰参数

装饰参数用于改变节点的执行行为：

| 参数   | 说明                |
| ---- | ----------------- |
| 结果取反 | 勾选后，成功变为失败，失败变为成功 |
| 失败重试 | 设置节点失败后的重试次数      |
| 超时时间 | 超时时间（毫秒）          |
| 检测间隔 | 检测间隔（毫秒）          |

## 9. 黑板系统

### 9.1 什么是黑板？

黑板是节点间通信的核心机制，采用观察者模式，允许节点间解耦通信。

### 9.2 内置变量

| 变量名                       | 说明                 |
| ------------------------- | ------------------ |
| last\_detection\_position | 最近一次检测到的位置坐标       |
| last\_number\_value       | 最近一次识别的数字值         |
| last\_extracted\_text     | 最近一次文本提取的内容        |
| last\_input\_text         | 最近一次文本输入的内容（需开启保存） |
| execution\_count          | 执行计数               |

### 9.3 使用方式

**写入变量**：

- 条件节点检测成功后自动写入位置
- 设变量节点主动设置变量值

**读取变量**：

- 点击节点使用"点击最近检测点"功能
- 变量判断节点判断变量值

### 9.4 典型通信场景

**场景：OCR检测 → 点击检测位置**

```
OCRConditionNode                    MouseClickNode
     │                                    │
     │ 1. 检测关键词                       │
     │ 2. blackboard.set(                 │
     │    "last_detection_position", pos) │
     │ ──────────────────────────────────►│
     │                                    │ 3. blackboard.get()
     │                                    │ 4. 点击该位置
```

1. 在节点属性面板中选择资源文件
2. 系统自动检测资源类型
3. 文件自动复制到项目对应目录
4. 自动生成相对路径引用

**资源类型映射**：

| 节点属性           | 资源类型   | 目标目录              |
| -------------- | ------ | ----------------- |
| template\_path | image  | images/templates/ |
| script\_path   | script | scripts/script/   |
| code\_path     | code   | scripts/code/     |
| sound\_path    | audio  | audio/alarms/     |
| file\_path     | data   | data/config/      |

### 11.5 项目导出

1. 点击工具栏【导出】按钮
2. 选择 ZIP 文件保存位置
3. 系统自动打包项目文件夹
4. 生成使用说明文档

**自动排除的文件**：

- Python 编译文件
- 缓存目录（__pycache__, cache/）
- 临时文件
- 备份文件（tree\_backup\_\*）

### 11.6 自动备份

- 每次保存项目时自动创建备份
- 备份文件命名：`tree_backup_YYYYMMDD_HHMMSS.json`
- 自动保留最近 5 个备份文件

## 14. 常见问题

### Q1: 如何实现循环执行？

**A**: 设置开始或组合节点的"重复次数"为 -1 即可实现无限循环。

### Q2: 点击位置不正确怎么办？

**A**:

1. 检查检测区域是否正确
2. 确保勾选了"点击最近检测点"
3. 检测节点会保存绝对坐标，点击节点会使用该坐标

### Q3: OCR识别不准确怎么办？

**A**:

1. 确保检测区域只包含需要识别的文字
2. 尝试更换 OCR 语言设置
3. 中文识别已针对中文特点优化

### Q4: 图像匹配失败怎么办？

**A**:

1. 降低匹配阈值（如从 90 降到 80）
2. 确保模板图像清晰
3. 检查检测区域是否正确

### Q5: 如何使用"点击最近检测点"？

**A**:

1. 在条件节点（OCR/图像/颜色）中检测目标
2. 检测成功后位置会自动保存到黑板
3. 在点击节点中勾选"点击最近检测点"
4. 点击时会使用最近一次检测到的位置

### Q6: 如何实现长按操作？

**A**: 在点击节点中设置：

- 动作：press
- 按住时长：500（毫秒）

### Q7: 如何实现拖拽操作？

**A**: 使用鼠标移动节点的拖拽功能：

1. 设置移动类型=拖拽
2. 设置位置为拖拽起点
3. 设置拖拽终点为目标位置
4. 设置拖拽时长（建议 500-1000 毫秒）

### Q8: 如何实现无限点击？

**A**: 在点击节点中设置：

- 点击次数：-1
- 点击间隔：100（毫秒）
- 节点会持续点击，直到按 Escape 键停止

### Q9: 节点执行顺序如何确定？

**A**:

- 连接线上会显示序号（1、2、3...）
- 按序号顺序执行
- 如需调整，删除连接线后重新按期望顺序连接

### Q10: 如何分享项目？

**A**:

1. 点击【导出】按钮导出为 ZIP 文件
2. 将 ZIP 文件分享给其他用户
3. 其他用户点击【导入】按钮，选中 ZIP 文件即可打开项目
