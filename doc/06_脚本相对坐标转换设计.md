# 脚本相对坐标转换功能设计

## 1. 功能概述

为脚本节点（ScriptNode）提供坐标转换功能，将录制时保存的屏幕绝对坐标转换为窗口相对坐标，以适配窗口绑定功能。

## 3. 脚本文件格式扩展

### 3.1 原格式

```
Delay 100
MoveTo 100, 200
LeftDown 1
LeftUp 1
```

### 3.2 转换后格式

```
# Window: 记事本 - 无标题
# WindowRect: 100, 200, 800, 600
Delay 100
MoveTo 50, 100
LeftDown 1
LeftUp 1
```

### 3.3 窗口标记说明

| 标记 | 说明 |
|------|------|
| `# Window: <窗口标题>` | 记录转换时的窗口标题关键字 |
| `# WindowRect: <left, top, right, bottom>` | 记录转换时的窗口矩形区域 |

### 3.4 脚本运行兼容性

`# Window:` 和 `# WindowRect:` 开头的行会被 `ScriptExecutor` 忽略（因为不是有效命令），不影响脚本运行。

## 5. 坐标转换逻辑

### 5.1 屏幕绝对坐标 → 窗口相对坐标

```
窗口相对坐标 = 屏幕绝对坐标 - 窗口左上角坐标

示例：
  屏幕绝对坐标: (500, 300)
  窗口矩形: (100, 50, 800, 600)
  窗口相对坐标: (500-100, 300-50) = (400, 250)
```

### 5.2 转换公式

```python
def screen_to_window(screen_x, screen_y, window_rect):
    """
    屏幕绝对坐标 → 窗口相对坐标
    
    Args:
        screen_x: 屏幕绝对 X 坐标
        screen_y: 屏幕绝对 Y 坐标
        window_rect: (left, top, right, bottom)
    
    Returns:
        (window_x, window_y): 窗口相对坐标
    """
    window_x = screen_x - window_rect[0]
    window_y = screen_y - window_rect[1]
    return window_x, window_y
```

## 6. 脚本备份功能

### 6.1 备份策略

| 策略 | 说明 |
|------|------|
| 备份时机 | 转换坐标前自动备份 |
| 备份位置 | 原脚本文件同目录 |
| 备份文件名 | `<原文件名>.bak` |
| 覆盖策略 | 如果 `.bak` 文件已存在，直接覆盖 |

### 6.2 备份文件示例

```
原文件: ./scripts/test.txt
备份文件: ./scripts/test.txt.bak
```

### 6.3 备份内容

备份文件保存的是转换前的原始脚本内容（屏幕绝对坐标）。

## 9. 注意事项

### 9.1 窗口位置变化

如果窗口移动了，之前转换的相对坐标仍然有效，因为相对坐标是相对于窗口左上角的。

### 9.2 多次转换保护

通过窗口标记防止重复转换，避免坐标被多次减去窗口偏移。

### 9.3 脚本运行兼容性

注释行（以 `#` 开头）会被 `ScriptExecutor` 自动忽略，不影响脚本运行。

### 9.4 备份文件管理

- 备份文件不会被自动删除
- 用户可以手动删除或使用备份文件恢复原脚本
