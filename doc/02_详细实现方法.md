# AutoDoor 行为树系统 — 详细实现方法

## 1 行为树节点类型实现

### 1.1 节点基类 (Node)

所有节点的抽象基类，定义通用接口和默认行为：

```python
class Node(ABC):
    NODE_TYPE = "Node"                    # 类级别节点类型标识
    
    def __init__(self, node_id=None, config=None):
        self.node_id = node_id or uuid4()[:8]   # 唯一标识
        self.config = config or NodeConfig()     # 节点配置
        self.status = NodeStatus.SUCCESS         # 当前状态
        self.children: List[Node] = []           # 子节点列表
        self.parent: Optional[Node] = None       # 父节点引用
        self._tick_count = 0                     # tick计数
    
    @abstractmethod
    def tick(self, context: ExecutionContext) -> NodeStatus:
        """核心方法：执行节点逻辑，返回执行状态"""
        pass
```

**设计要点：**

- 使用 `uuid4()[:8]` 生成短 ID，兼顾唯一性与可读性
- `config` 使用 `NodeConfig` 数据类，采用扁平化结构存储节点参数（与原项目兼容）
- `children` 列表维护子节点有序集合
- `tick()` 是纯抽象方法，子类必须实现
- `reset(reset_counters=True)` 方法支持控制是否重置计数器，用于 retry/repeat 场景

**节点重置机制：**

节点重置是行为树状态管理的核心，需要区分不同场景：

| 场景   | 方法                           | 行为                          |
| ---- | ---------------------------- | --------------------------- |
| 完全重置 | `reset(reset_counters=True)` | 重置所有状态和计数器                  |
| 重试重置 | `_reset_for_retry()`         | 设置 `status=RUNNING`，保留重试计数器 |
| 重复重置 | `_reset_for_repeat()`        | 调用 `reset(False)`，保留重复计数器   |

**重要：** `_reset_for_retry()` 和 `_reset_for_repeat()` 必须将 `status` 设置为 `RUNNING`，确保节点在重试/重复期间返回正确状态。

**`_reset_for_retry()`** **完整实现（V1.3.2b 修复）：**

```python
def _reset_for_retry(self) -> None:
    """重试时重置状态（保留重试计数器）
    
    V1.3.2b 修复：必须重置所有执行状态字段，否则会导致：
    - ConditionNode 跳过条件检查（_children_running 未重置）
    - SequenceNode 从错误索引继续（current_index 未重置）
    - ConditionNode 返回缓存状态（_last_check_time 未正确重置）
    """
    self.status = NodeStatus.RUNNING
    self._tick_count = 0
    self._start_time = None
    self._child_index = 0
    self._children_running = False
    
    # CompositeNode 子类字段
    if hasattr(self, 'current_index'):
        self.current_index = 0
    if hasattr(self, '_last_child_finish_time'):
        self._last_child_finish_time = None
    
    # 递归重置所有子节点
    for child in self.children:
        child.reset()
```

**各组合节点的** **`_reset_for_retry()`** **覆盖：**

| 节点类型         | 额外重置字段                                                                                                               |
| ------------ | -------------------------------------------------------------------------------------------------------------------- |
| SequenceNode | `_last_child_finish_time = None`                                                                                     |
| SelectorNode | `_last_child_time = 0`                                                                                               |
| ParallelNode | `cached_statuses.clear()`                                                                                            |
| RandomNode   | `_executed_indices.clear()`, `_success_indices.clear()`, `_failure_indices.clear()`, `_current_running_index = None` |

**ConditionNode.reset() 修复（V1.3.2b）：**

```python
def reset(self, reset_counters: bool = True) -> None:
    super().reset(reset_counters)
    # 关键：必须与构造函数一致，确保首次 tick 执行条件检查
    self._last_check_time = -self.check_interval_ms - 1  # 而非 0
```

### 1.2 复合节点 (CompositeNode)

复合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 1.2.1 顺序节点 (SequenceNode)

**语义：** 按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

**支持参数：**

| 参数                    | 说明             | 默认值   |
| --------------------- | -------------- | ----- |
| continue\_on\_failure | 失败后是否继续执行后续子节点 | False |
| childinterval         | 子节点执行间隔（毫秒）    | 0     |
| childinterval\_random | 子节点间隔随机范围（±毫秒） | 0     |

```
执行流程:
  child[0] ──SUCCESS──► child[1] ──SUCCESS──► child[2] ──SUCCESS──► return SUCCESS
     │                      │                      │
   FAILURE                FAILURE                FAILURE
     │                      │                      │
     ▼                      ▼                      ▼
  return FAILURE        return FAILURE         return FAILURE
  
  (若 continue_on_failure=True，失败后继续执行下一个子节点)
```

**实现伪代码：**

```
function SequenceNode.tick(context):
    if children is empty:
        return SUCCESS
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING          // 等待当前子节点完成
        
        if status == FAILURE:
            if continue_on_failure:
                current_index += 1  // 继续执行下一个子节点
                continue
            current_index = 0       // 重置，下次从头开始
            return FAILURE          // 短路：任一失败即失败
        
        // 子节点执行成功，添加间隔
        if childinterval > 0:
            actual_interval = get_random_duration(childinterval, childinterval_random)
            sleep(actual_interval / 1000.0)
        
        current_index += 1          // 成功则继续下一个
    
    current_index = 0               // 全部成功，重置
    return SUCCESS
```

**关键实现细节：**

- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 失败时重置 `current_index = 0`，确保下次 tick 从头开始
- 跳过 `enabled = False` 的子节点
- `continue_on_failure=True` 时，失败后继续执行后续子节点
- `childinterval` 支持在子节点之间添加延迟，模拟更自然的操作

#### 1.2.2 选择节点 (SelectorNode)

**语义：** 按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

**支持参数：**

| 参数                    | 说明             | 默认值 |
| --------------------- | -------------- | --- |
| childinterval         | 子节点执行间隔（毫秒）    | 0   |
| childinterval\_random | 子节点间隔随机范围（±毫秒） | 0   |

```
执行流程:
  child[0] ──FAILURE──► child[1] ──FAILURE──► child[2] ──FAILURE──► return FAILURE
     │                      │                      │
   SUCCESS                SUCCESS                SUCCESS
     │                      │                      │
     ▼                      ▼                      ▼
  return SUCCESS         return SUCCESS          return SUCCESS
```

**实现伪代码：**

```
function SelectorNode.tick(context):
    if children is empty:
        return FAILURE
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING
        
        if status == SUCCESS:
            current_index = 0       // 重置
            return SUCCESS          // 短路：任一成功即成功
        
        // 子节点执行失败，添加间隔后尝试下一个
        if childinterval > 0:
            actual_interval = get_random_duration(childinterval, childinterval_random)
            sleep(actual_interval / 1000.0)
        
        current_index += 1          // 失败则尝试下一个
    
    current_index = 0
    return FAILURE                  // 全部失败
```

#### 1.2.3 并行节点 (ParallelNode)

**语义：** 同时执行所有子节点，根据成功策略决定返回条件。

**成功策略：**

- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**实现伪代码：**

```
function ParallelNode.tick(context):
    if children is empty:
        return SUCCESS
    
    success_count = 0
    failure_count = 0
    running_children = []
    
    for i, child in enumerate(children):
        if not child.config.enabled:
            continue
        
        // 检查缓存状态（已完成的子节点不再重复执行）
        if i in cached_statuses:
            status = cached_statuses[i]
            if status == SUCCESS: success_count += 1; continue
            if status == FAILURE: failure_count += 1; continue
        
        status = child.tick(context)
        
        if status == SUCCESS:
            cached_statuses[i] = SUCCESS
            success_count += 1
        elif status == FAILURE:
            cached_statuses[i] = FAILURE
            failure_count += 1
        elif status == RUNNING:
            running_children.append(child)
    
    // require_one 策略：任一成功即返回
    if success_policy == REQUIRE_ONE and success_count > 0:
        abort_running(context, running_children)
        return SUCCESS
    
    // 仍有运行中的子节点
    if running_children:
        return RUNNING
    
    // 所有子节点执行完毕
    if success_policy == REQUIRE_ALL:
        return SUCCESS if success_count == enabled_count else FAILURE
    else:
        return SUCCESS if success_count > 0 else FAILURE
```

**关键实现细节：**

- `cached_statuses` 缓存已完成子节点的状态，避免重复 tick
- `require_one` 模式下，一旦有子节点成功，立即中止其他运行中的子节点
- `reset()` 时清空缓存
- `_reset_for_retry()` 和 `_reset_for_repeat()` 时也清空缓存，确保重试/重复执行时重新评估所有子节点

**日志输出实现：**

```python
def _tick_internal(self, context: ExecutionContext) -> NodeStatus:
    from bt_utils.log_manager import LogManager
    
    if not self.children:
        LogManager.instance().log_success(
            node_type="并行节点",
            node_name=self.name
        )
        return NodeStatus.SUCCESS
    
    # ... 执行逻辑 ...
    
    if success_count == enabled_count:
        LogManager.instance().log_success(
            node_type="并行节点",
            node_name=self.name
        )
        return NodeStatus.SUCCESS
    else:
        LogManager.instance().log_failure(
            node_type="并行节点",
            node_name=self.name,
            reason=f"成功 {success_count}/{enabled_count} 个子节点"
        )
        return NodeStatus.FAILURE
```

#### 1.2.4 随机节点 (RandomNode)

**语义：** 随机执行子节点，根据成功策略决定返回条件。

**成功策略：**

- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**参数配置：**

| 参数                    | 说明                              | 默认值            |
| --------------------- | ------------------------------- | -------------- |
| success\_policy       | 成功策略（require\_all/require\_one） | "require\_all" |
| continue\_on\_failure | 失败后是否继续执行（仅 require\_all 策略生效）  | False          |
| fully\_random         | 完全随机模式（True 时已执行的子节点仍可被选中）      | False          |

**执行流程：**

```
执行流程（非完全随机模式）:
  随机池: [child[0], child[1], child[2], ...]
      │
      ▼
  随机选择 child[i] 执行
      │
      ├── SUCCESS: 记录成功，从随机池移除
      │   └── require_one 策略: 立即返回 SUCCESS
      │
      └── FAILURE: 记录失败，从随机池移除
          └── continue_on_failure=False 且 require_all 策略: 立即返回 FAILURE
      │
      ▼
  随机池为空？
      │
      ├── 是: 根据成功策略返回最终状态
      └── 否: 继续随机选择执行
```

**实现伪代码：**

```
function RandomNode.tick(context):
    if children is empty:
        log_success(node_type="随机节点", node_name=self.name)
        return SUCCESS
    
    // 初始化启用的子节点索引
    if enabled_indices is empty:
        enabled_indices = [i for i, child in enumerate(children) if child.config.enabled]
    
    if enabled_indices is empty:
        log_success(node_type="随机节点", node_name=self.name)
        return SUCCESS
    
    // 恢复运行中的子节点
    if current_running_index is not None:
        child = children[current_running_index]
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING
        
        if status == SUCCESS:
            success_indices.add(current_running_index)
            if success_policy == REQUIRE_ONE:
                log_success(node_type="随机节点", node_name=self.name)
                return SUCCESS
        elif status == FAILURE:
            failure_indices.add(current_running_index)
            if not continue_on_failure and success_policy == REQUIRE_ALL:
                log_failure(node_type="随机节点", node_name=self.name, 
                           reason=f"子节点 '{child.name}' 执行失败")
                return FAILURE
        
        current_running_index = None
    
    // 获取可执行的子节点索引
    available_indices = get_available_indices()
    
    if available_indices is empty:
        return determine_final_status()
    
    // 完全随机模式：所有子节点都执行过一次后结束
    if fully_random and executed_indices == set(enabled_indices):
        return determine_final_status()
    
    // 随机选择下一个子节点
    next_index = random.choice(available_indices)
    executed_indices.add(next_index)
    current_running_index = next_index
    
    child = children[next_index]
    status = child.tick(context)
    
    if status == RUNNING:
        return RUNNING
    
    if status == SUCCESS:
        success_indices.add(next_index)
        if success_policy == REQUIRE_ONE:
            log_success(node_type="随机节点", node_name=self.name)
            return SUCCESS
    elif status == FAILURE:
        failure_indices.add(next_index)
        if not continue_on_failure and success_policy == REQUIRE_ALL:
            log_failure(node_type="随机节点", node_name=self.name,
                       reason=f"子节点 '{child.name}' 执行失败")
            return FAILURE
    
    current_running_index = None
    
    // 检查是否还有可执行的子节点
    available_indices = get_available_indices()
    if available_indices is empty:
        return determine_final_status()
    
    return RUNNING

function get_available_indices():
    if fully_random:
        return enabled_indices  // 完全随机：所有启用的子节点都可被选中
    else:
        return [i for i in enabled_indices if i not in executed_indices]  // 非完全随机：已执行的不再被选中

function determine_final_status():
    enabled_count = len(enabled_indices)
    success_count = len(success_indices)
    
    if success_policy == REQUIRE_ALL:
        if success_count == enabled_count:
            log_success(node_type="随机节点", node_name=self.name)
            return SUCCESS
        else:
            log_failure(node_type="随机节点", node_name=self.name,
                       reason=f"成功 {success_count}/{enabled_count} 个子节点")
            return FAILURE
    else:
        if success_count > 0:
            log_success(node_type="随机节点", node_name=self.name)
            return SUCCESS
        else:
            log_failure(node_type="随机节点", node_name=self.name,
                       reason="所有子节点都执行失败")
            return FAILURE
```

**关键实现细节：**

- `_executed_indices`：记录已执行的子节点索引（用于非完全随机模式）
- `_success_indices`：记录成功的子节点索引
- `_failure_indices`：记录失败的子节点索引
- `_current_running_index`：当前正在执行的子节点索引（用于恢复 RUNNING 状态）
- `fully_random=True` 时，子节点可被重复选中，直到所有子节点都执行过一次
- `continue_on_failure` 仅在 `require_all` 策略下生效

#### 1.2.5 开始节点 (StartNode)

**语义：** 作为行为树的根节点，顺序执行所有子节点，但与 SequenceNode 不同的是，子节点失败后会继续执行后续子节点。

**继承关系：** StartNode 继承自 CompositeNode，使用 `current_index` 实现顺序执行逻辑。

```
执行流程:
  child[0] ──SUCCESS/FAILURE──► child[1] ──SUCCESS/FAILURE──► child[2] ──► ...
     │                              │                              │
     └──────────────────────────────┴──────────────────────────────┘
                    所有子节点执行完毕后返回
     │
     ▼
  if has_failure: return FAILURE
  else: return SUCCESS
```

**与 SequenceNode 的区别：**

| 特性   | SequenceNode      | StartNode         |
| ---- | ----------------- | ----------------- |
| 失败处理 | 任一失败立即返回          | 失败后继续执行           |
| 执行模式 | 短路执行              | 全部执行              |
| 用途   | 条件序列              | 行为树根节点            |
| 状态记忆 | 使用 current\_index | 使用 current\_index |

**实现伪代码：**

```
function StartNode.tick(context):
    if children is empty:
        log_success(node_type="开始节点", node_name=self.name)
        return SUCCESS
    
    has_failure = false
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING    // 等待当前子节点完成
        
        if status == FAILURE:
            has_failure = true
        // SUCCESS: 继续执行下一个子节点
        
        current_index += 1
    
    // 所有子节点执行完毕，重置索引
    current_index = 0
    
    // 注意：StartNode 始终返回 SUCCESS，即使有子节点失败
    // 这确保行为树可以继续执行，不会因单个节点失败而中断
    log_success(node_type="开始节点", node_name=self.name)
    return SUCCESS
```

**关键实现细节：**

- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 子节点失败时设置 `has_failure = true`，但继续执行后续子节点
- 所有子节点执行完毕后重置 `current_index = 0`
- 跳过 `enabled = False` 的子节点
- **始终返回 SUCCESS**：即使有子节点失败，StartNode 也返回 SUCCESS，确保行为树可以继续执行

### 1.3 条件节点 (ConditionNode)

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

**基类实现：**

```python
class ConditionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.invert = self.config.get("invert", False)           # 结果取反
        self.check_interval_ms = self.config.get("check_interval_ms", 300)  # 检测间隔
        self._last_check_time = 0
        self._children_running = False  # 子节点执行状态追踪标志
        
        # 坐标偏移参数 (向后兼容：支持旧的 offset_x/offset_y 格式)
        offset = self.config.get("offset", None)
        if offset is not None:
            if isinstance(offset, (list, tuple)) and len(offset) >= 2:
                self.offset = (int(offset[0]), int(offset[1]))
            else:
                self.offset = (0, 0)
        else:
            # 向后兼容：从旧的 offset_x/offset_y 读取
            offset_x = self.config.get_int("offset_x", 0)
            offset_y = self.config.get_int("offset_y", 0)
            self.offset = (offset_x, offset_y)
    
    def _apply_offset(self, position: tuple) -> tuple:
        """应用坐标偏移"""
        if position is None:
            return None
        return (position[0] + self.offset[0], position[1] + self.offset[1])
    
    def tick(self, context):
        // 检查子节点是否正在执行
        if self._children_running and self.children:
            return self._execute_children(context)
        
        // 检测间隔控制
        current_time = context.elapsed_time * 1000
        if current_time - self._last_check_time < self.check_interval_ms:
            return self.status    // 未到检测时间，返回上次状态
        
        self._last_check_time = current_time
        result = self._check_condition(context)   // 子类实现
        
        if self.invert:
            result = not result
        
        status = SUCCESS if result else FAILURE
        self.status = status
        
        // 条件成功时执行子节点
        if status == SUCCESS and self.children:
            # GUI状态通知（详见GUI状态通知机制段落）
            self._children_running = True
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _check_condition(self, context) -> bool:
        pass
    
    # 通用辅助方法（已提取到基类）
    def _validate_region(self, region) -> bool:
        """验证区域配置是否有效"""
        if region is None:
            return True
        if isinstance(region, (list, tuple)) and len(region) == 4:
            return all(isinstance(x, int) for x in region)
        return False
    
    def _get_region_image(self, context):
        """获取区域图像（截图+裁剪）"""
        try:
            region_mode = self.config.get("region_mode", "fixed")
            if region_mode == "dynamic":
                region = self._resolve_dynamic_region(context)
            else:
                region = self._parse_region(self.config.get("region", None))
            // 差异截图优化（V1.5.2a 新增）：无region时使用上次检测位置附近区域
            if region is None:
                region = self._get_fallback_region(context)
            return context.get_screenshot(region)
        except Exception as e:
            from bt_utils.exception_handler import log_exception
            log_exception(e, f"{self.NODE_TYPE} '{self.name}' 截图失败")
            return None
    
    def _resolve_dynamic_region(self, context):
        """根据黑板锚点 + 偏移量计算动态区域"""
        use_last_pos = self.config.get_bool("region_use_last_pos", True)
        if use_last_pos:
            anchor_key = "last_detection_position"
        else:
            anchor_key = self.config.get("region_anchor", "") or "last_detection_position"

        pos = context.blackboard.get(anchor_key)
        if pos is None or not isinstance(pos, (tuple, list)) or len(pos) < 2:
            from bt_utils.log_manager import LogManager
            LogManager.instance().log_failure(
                node_type=self.NODE_TYPE,
                node_name=self.name,
                reason=f"动态区域锚点 '{anchor_key}' 不存在或格式无效，使用全屏截图"
            )
            return None

        cx, cy = int(pos[0]), int(pos[1])
        offset = self.config.get("region_offset", [-50, -50, 50, 50])
        if isinstance(offset, str):
            try:
                offset = [int(x.strip()) for x in offset.split(",")]
            except (ValueError, AttributeError):
                offset = [-50, -50, 50, 50]
        if not isinstance(offset, (list, tuple)) or len(offset) < 4:
            offset = [-50, -50, 50, 50]

        return (cx + int(offset[0]), cy + int(offset[1]),
                cx + int(offset[2]), cy + int(offset[3]))
    
    def _log_condition_result(self, success, reason=None, extra_info=None):
        """记录条件检测结果日志"""
        from bt_utils.log_manager import LogManager
        if success:
            LogManager().log_success(node_type=self.NODE_TYPE, node_name=self.name)
        else:
            log_reason = reason or "条件不满足"
            if extra_info:
                log_reason += f"，{extra_info}"
            LogManager().log_failure(node_type=self.NODE_TYPE, node_name=self.name, reason=log_reason)
    
    def _save_position(self, context, position):
        """保存检测位置到黑板（自动拆分 x/y 坐标）"""
        if position and hasattr(context, 'blackboard'):
            context.blackboard.set(self.position_key, position)
            context.blackboard.set("last_detection_x", position[0])
            context.blackboard.set("last_detection_y", position[1])

    // V1.5.2a 新增：智能跳过机制
    _STABLE_THRESHOLD_LOW = 5    // 连续5次相同结果
    _STABLE_THRESHOLD_HIGH = 15  // 连续15次相同结果

    def _should_skip_check(self) -> bool:
        """连续多次检测结果相同时，自动增大检测间隔"""
        if self._consecutive_same_count >= self._STABLE_THRESHOLD_HIGH:
            return True  // 间隔×4
        if self._consecutive_same_count >= self._STABLE_THRESHOLD_LOW:
            return True  // 间隔×2
        return False

    // V1.5.2a 新增：差异截图回退区域
    _DEFAULT_REGION_RADIUS = 150

    def _get_fallback_region(self, context):
        """当未配置region时，使用上次检测位置附近的区域"""
        pos = context.blackboard.get("last_detection_position")
        if pos is None or not isinstance(pos, (tuple, list)) or len(pos) < 2:
            return None  // 无可用位置，降级全屏截图
        cx, cy = int(pos[0]), int(pos[1])
        r = self._DEFAULT_REGION_RADIUS
        return (max(0, cx - r), max(0, cy - r), cx + r, cy + r)
```

**动态区域检测：**

条件节点支持两种区域选择方式，通过 `region_mode` 配置切换：

- **固定区域检测**（`"fixed"`，默认）：使用 `region` 字段指定的固定坐标区域，与改造前行为一致
- **动态区域检测**（`"dynamic"`）：根据黑板锚点位置 + 偏移量动态计算检测区域

动态区域计算流程：

```
1. 读取 region_mode 配置
   ├─ "fixed" → 使用 _parse_region 解析固定区域
   └─ "dynamic" → 调用 _resolve_dynamic_region

2. _resolve_dynamic_region 执行：
   a. 确定锚点变量名
      - region_use_last_pos=True → anchor_key="last_detection_position"
      - region_use_last_pos=False → anchor_key=config.get("region_anchor", "")
      - anchor_key 为空 → 回退到 "last_detection_position"
   b. 从黑板读取锚点位置
      - 锚点不存在 → 输出警告日志，返回 None（全屏截图）
   c. 读取偏移量 region_offset（支持字符串和列表格式）
   d. 计算动态区域：
      region = (anchor_x + offset_x1, anchor_y + offset_y1,
                anchor_x + offset_x2, anchor_y + offset_y2)

3. 返回计算后的区域 → context.get_screenshot(region)
```

#### 1.3.5 变量判断节点 (VariableConditionNode)

**语义：** 判断黑板变量是否满足指定条件，支持常量值比较、变量间比较和存在性判断。

**支持参数：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| variable_name | 要判断的黑板变量名 | "" |
| operator | 比较运算符 | "==" |
| compare_type | 比较值类型（常量值/变量名） | "constant" |
| compare_value | 比较值（常量值模式时显示） | "" |
| compare_variable | 比较变量（变量名模式时显示） | "" |

**运算符中文显示映射：**

| 中文显示 | 内部值 | 说明 |
|---------|--------|------|
| 等于 | == | 值相等 |
| 不等于 | != | 值不等 |
| 大于 | > | 数值大于 |
| 大于等于 | >= | 数值大于等于 |
| 小于 | < | 数值小于 |
| 小于等于 | <= | 数值小于等于 |
| 存在 | exists | 变量存在性检查 |
| 不存在 | not_exists | 变量不存在检查 |

**字段条件隐藏规则：**

| 运算符 | 比较值类型 | 显示的字段 | 隐藏的字段 |
|--------|-----------|-----------|-----------|
| 等于/不等于/大于等 | 常量值 | 变量名、运算符、比较值类型、比较值 | 比较变量 |
| 等于/不等于/大于等 | 变量名 | 变量名、运算符、比较值类型、比较变量 | 比较值 |
| 存在/不存在 | — | 变量名、运算符 | 比较值类型、比较值、比较变量 |

**执行流程：**

```
执行流程:
  1. 若 operator == "exists":
       → blackboard.exists(variable_name) → 返回布尔值
  2. 若 operator == "not_exists":
       → not blackboard.exists(variable_name) → 返回布尔值
  3. blackboard.get(variable_name) → 获取变量当前值
     若值为 None → 返回 False（变量不存在）
  4. 若 compare_type == "constant":
       → _parse_value(compare_value) → 解析比较值
  5. 若 compare_type == "variable":
       → blackboard.get(compare_variable) → 获取比较变量值
       若值为 None → 返回 False（比较变量不存在或值为 None）+ 日志提示
  6. _compare_value(current_value, operator, target_value) → 比较运算
  7. 支持运算符: ==, !=, >, >=, <, <=, exists, not_exists
```

**关键实现细节：**

- `compare_type == "variable"` 时，比较值从黑板动态获取，保留原始类型（如 tuple、int），不经字符串转换
- 比较变量不存在或值为 None 时，显式返回 False 并输出日志「比较变量 '{compare_var}' 不存在或值为 None」，避免将 None 传入 `_compare_value` 导致 TypeError
- `exists/not_exists` 运算符仅调用 `blackboard.exists()`，不获取变量值，效率更高
- `_parse_value()` 方法支持将字符串 "true"/"false"/"none" 解析为 Python 布尔值/None，以及整数和浮点数自动类型推断

```
function TextExtractNode._check_condition(context):
    screenshot = _get_region_image(context)
    if screenshot is None:
        return False
    
    all_text = OCRManager.get_all_text(screenshot, language, preprocess_mode)
    if not all_text:
        log_condition_result(False, "未识别到文本")
        return False
    
    if extract_mode == "all":
        extracted_text = all_text
    else:
        extracted_text = _extract_keywords_text(all_text, keywords)
    
    blackboard.set(output_key, extracted_text)
    
    if save_all_text:
        blackboard.set(all_text_key, all_text)
    
    if save_position and region:
        center = ((region[0]+region[2])//2, (region[1]+region[3])//2)
        blackboard.set(position_key, center)
    
    if extracted_text:
        log_condition_result(True)
        return True
    else:
        log_condition_result(False, "未提取到匹配的文本")
        return False
```

**关键词提取逻辑：**

```
function _extract_keywords_text(all_text, keywords):
    lines = all_text.split('\n')
    matched_lines = []
    for line in lines:
        line = line.strip()
        if line and keywords in line:
            matched_lines.append(line)
    return '\n'.join(matched_lines)
```

**关键实现细节：**

- 继承 `ConditionNode`，使用 `_check_condition()` 实现检测逻辑
- 使用 `_get_region_image()` 获取区域截图（继承自 ConditionNode）
- 使用 `_log_condition_result()` 输出检测日志（继承自 ConditionNode）
- `output_key` 支持自定义变量名，默认为 `last_extracted_text`，与 TextInputNode 的默认 `blackboard_key` 一致
- `save_all_text` 可选保存完整 OCR 识别结果，用于需要原始文本的场景
- `save_position` 保存检测区域中心坐标，与 OCRConditionNode 等条件节点保持一致

### 1.4 动作节点 (ActionNode)

动作节点执行具体操作，产生副作用。

**基类实现：**

```python
class ActionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self._children_running = False  # 子节点执行状态追踪标志
    
    def tick(self, context):
        // 检查子节点是否正在执行
        if self._children_running and self.children:
            return self._execute_children(context)
        
        status = self._execute_action(context)   // 子类实现
        self.status = status
        
        // 动作成功后执行子节点
        if status == SUCCESS and self.children:
            // 通知 GUI 节点状态为 SUCCESS
            context.notify_node_status(self.node_id, "success")
            self._children_running = True
            return self._execute_children(context)
        
        return status
    
    @abstractmethod
    def _execute_action(self, context) -> NodeStatus:
        pass
```

**子节点执行状态追踪：**

ActionNode 使用 `_children_running` 标志追踪子节点的执行状态：

| 属性                  | 说明               |
| ------------------- | ---------------- |
| `_children_running` | 布尔标志，标记子节点是否正在执行 |

**状态追踪流程：**

```
ActionNode 执行流程:
┌─────────────────────────────────────────────────────────────────────┐
│  1. 首次 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 False，执行动作                                            │
│     - 若动作成功且有子节点:                                           │
│       - 通知 GUI 节点状态为 SUCCESS                                   │
│       - 设置 _children_running = True                                │
│       - 开始执行子节点                                                │
│                                                                      │
│  2. 后续 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 True，跳过动作执行，继续执行子节点                          │
│     - 子节点执行完成后，重置 _children_running = False                │
└─────────────────────────────────────────────────────────────────────┘
```

**GUI 状态通知机制：**

当 ActionNode 或 ConditionNode 成功执行后需要执行子节点时，必须先通知 GUI 节点状态为 SUCCESS，确保界面正确显示节点状态：

```python

    context.notify_node_status(self.node_id, "success")
    self._children_running = True
    return self._execute_children(context)
```

**随机值计算函数：**

```python
def get_random_value(base_value: int, random_range: int = 0, min_value: int = None) -> int:
    """
    计算带随机范围的值
    
    Args:
        base_value: 基础值
        random_range: 随机范围（±）
        min_value: 最小值限制（可选）
    
    Returns:
        计算后的随机值
    """
    if random_range <= 0:
        return base_value
    
    min_val = base_value - random_range
    max_val = base_value + random_range
    
    if min_value is not None:
        min_val = max(min_val, min_value)
    
    return random.randint(int(min_val), int(max_val))
```

| -------------- | -------------------------- |

| > 0            | 使用 smoothstep 缓动函数平滑移动指定时长 |

**GUI字段条件显示：**

- 当操作类型为"移动"时，隐藏：拖拽按键
- 当开启"增量移动"时，隐藏：终点、终点使用最近检测点、终点黑板变量名

```python
class MouseMoveNode(ActionNode):
    def _smoothstep(self, t: float) -> float:
        """Smoothstep 缓动函数"""
        return t * t * (3 - 2 * t)
    
    def _execute_action(self, context) -> NodeStatus:
        # 获取起点位置
        start_pos = self._get_start_position(context)
        
        # 获取终点位置
        if self.relative:
            # 增量移动：终点 = 起点 + 增量值
            end_pos = (start_pos[0] + self.offset[0], start_pos[1] + self.offset[1])
        else:
            end_pos = self._get_end_position(context)
        
        if self.move_type == "拖拽":
            return self._execute_drag(context, start_pos, end_pos)
        else:
            return self._execute_move(context, start_pos, end_pos)
    
    def _execute_move(self, context, start_pos, end_pos) -> NodeStatus:
        # 移动到起点
        context.execute_mouse_move(start_pos, relative=False)
        
        if not end_pos:
            return NodeStatus.SUCCESS
        
        # 计算实际移动时长
        actual_duration = get_random_duration(self.move_duration, self.move_duration_random)
        
        if actual_duration > 0:
            # 平滑移动到终点
            total_duration_sec = actual_duration / 1000.0
            steps = max(10, actual_duration // 50)
            dx = end_pos[0] - start_pos[0]
            dy = end_pos[1] - start_pos[1]
            
            start_time = time.time()
            
            for i in range(steps):
                elapsed = time.time() - start_time
                progress = min(elapsed / total_duration_sec, 1.0)
                
                t = self._smoothstep(progress)
                current_x = int(start_pos[0] + dx * t)
                current_y = int(start_pos[1] + dy * t)
                context.execute_mouse_move((current_x, current_y), relative=False)
                
                if progress >= 1.0:
                    break
                
                time.sleep(total_duration_sec / steps)
        else:
            # 瞬间移动到终点
            context.execute_mouse_move(end_pos, relative=False)
        
        return NodeStatus.SUCCESS
    
    def _execute_drag(self, context, start_pos, end_pos) -> NodeStatus:
        # 移动到起点
        context.execute_mouse_move(start_pos, relative=False)
        
        # 按下鼠标
        context.execute_mouse_click(self.drag_button, start_pos, "down", 0)
        
        # 执行拖拽（与移动相同的平滑移动逻辑）
        actual_duration = get_random_duration(self.move_duration, self.move_duration_random)
        
        if actual_duration > 0:
            # 平滑拖拽到终点
            # ... 与移动相同的实现 ...
        else:
            # 瞬间移动到终点
            context.execute_mouse_move(end_pos, relative=False)
        
        # 释放鼠标
        context.execute_mouse_click(self.drag_button, end_pos, "up", 0)
        
        return NodeStatus.SUCCESS
```

```python
        self.sound_path = self.config.get("sound_path", "")
        self.volume = self.config.get_int("volume", 70)
        self._abort_flag = False

    def _execute_action(self, context) -> NodeStatus:
        # 1. 获取 Tab 管理器
        tab_manager = context.get_tab_manager()
        if not tab_manager:
            return NodeStatus.SUCCESS

        # 2. 查找目标 Tab
        target_name = self.config.get("target_tree", "")
        target_tab_id = self._find_tab_by_name(tab_manager, target_name)
        if not target_tab_id:
            return NodeStatus.SUCCESS

        # 3. 检查目标是否已在运行
        target_instance = tab_manager.get_tab(target_tab_id)
        if target_instance and target_instance.is_running:
            return NodeStatus.SUCCESS

        # 4. 播放启动音效
        self._play_sound(context)

        # 5. 启动目标行为树（跳过默认音效）
        tab_manager.start_tab(target_tab_id, skip_sound=True)

        return NodeStatus.SUCCESS
```

**参数设计说明：**

| 参数 | 说明 | 来源 |
|------|------|------|
| target_tree | 目标行为树名称 | 节点配置（从已加载 Tab 列表选择） |
| sound_path | 启动音效文件路径 | 节点配置（留空使用默认启动音效） |
| volume | 音量 (0-100) | 节点配置，默认 70 |

**关键设计点：**

- `SKIP_WINDOW_SWITCH = True`：启动树节点不触发窗口切换
- 通过 `context.get_tab_manager()` 获取 Tab 管理器
- `start_tab(skip_sound=True)` 跳过默认启动音效，避免与节点音效重复
- 音效播放使用 `AlarmPlayer.play()` + Sound 通道，`wait_complete=False` 异步播放

#### 1.4.12 停止树节点 (StopTreeNode)

StopTreeNode 是一个动作节点，用于停止当前行为树或其他已加载的行为树：

```python
class StopTreeNode(ActionNode):
    NODE_TYPE = "StopTreeNode"
    SKIP_WINDOW_SWITCH = True  # 不触发窗口切换

    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.target_tree = self.config.get("target_tree", "")
        self.sound_path = self.config.get("sound_path", "")
        self.volume = self.config.get_int("volume", 70)
        self._abort_flag = False

    def _execute_action(self, context) -> NodeStatus:
        # 1. 获取 Tab 管理器
        tab_manager = context.get_tab_manager()
        if not tab_manager:
            return NodeStatus.SUCCESS

        # 2. 确定目标 Tab（留空则停止当前行为树）
        target_name = self.config.get("target_tree", "")
        if not target_name:
            target_tab_id = context.get_current_tab_id()
        else:
            target_tab_id = self._find_tab_by_name(tab_manager, target_name)

        if not target_tab_id:
            return NodeStatus.SUCCESS

        # 3. 检查目标是否在运行
        target_instance = tab_manager.get_tab(target_tab_id)
        if not target_instance or not target_instance.is_running:
            return NodeStatus.SUCCESS

        # 4. 播放停止音效
        self._play_sound(context)

        # 5. 停止目标行为树（跳过默认音效）
        tab_manager.stop_tab(target_tab_id, skip_sound=True)

        return NodeStatus.SUCCESS
```

**参数设计说明：**

| 参数 | 说明 | 来源 |
|------|------|------|
| target_tree | 目标行为树名称 | 节点配置（留空停止当前行为树） |
| sound_path | 停止音效文件路径 | 节点配置（留空使用默认停止音效） |
| volume | 音量 (0-100) | 节点配置，默认 70 |

**关键设计点：**

- `target_tree` 为空时，通过 `context.get_current_tab_id()` 获取当前 Tab ID，停止当前行为树
- 停止音效在停止前播放（`stop_tab` 之前），因为停止后可能影响音效播放
- `stop_tab(skip_sound=True)` 跳过默认停止音效

**DelayNode 特殊实现：** DelayNode 是唯一可能返回 RUNNING 状态的动作节点。它使用 `_start_time` 记录开始时间，在 `duration_ms` 内持续返回 RUNNING，超时后返回 SUCCESS 并重置 `_start_time`。

**MouseMoveNode 移动时长实现：** MouseMoveNode 通过 `move_duration` 参数控制移动时长。当 `move_duration > 0` 时，使用 smoothstep 缓动函数实现平滑移动效果，将移动过程分解为多个小步，每步使用 smoothstep 插值计算位置，使鼠标移动更加自然流畅。当 `move_duration = 0` 时，鼠标瞬间移动到目标位置。

**MouseScrollNode 滚动实现（v2.0增强版）：** MouseScrollNode 支持 `distance`（滚动距离）、`clicks`（点击次数）、`direction`（滚动方向）三个参数，与旧项目保持兼容。

**DD版本滚轮特殊处理：**

由于DD64.dll的`DD_whl()`API限制（只接受1=向上/向前、2=向下/向后的离散方向码），DD版本的鼠标滚轮实现采用以下策略：

**核心算法：**

```python
# dd_input.py - mouse_scroll() 方法
def mouse_scroll(self, amount, position=None):
    # 1. 启动前检查停止标志（防止clicks循环重复启动）
    if self._check_stop():
        return
    
    # 2. 清除停止标志，开始新的滚动操作
    DDVirtualInput.clear_stop()
    
    # 3. 智能参数计算：最快速度 + 1秒超时保护
    MAX_DURATION = 1.0  # 单次滚动最长执行时间
    total = abs(amount)
    interval = MAX_DURATION / total  # 动态间隔
    
    # 4. 带中断检测的滚动循环
    for i in range(total):
        # 检查A: 是否超过1秒？
        if elapsed >= MAX_DURATION:
            break
            
        # 检查B: 是否收到F12停止请求？
        if self._check_stop():
            break
            
        # 执行滚动
        DD_whl(1 or 2)  # 方向码
        
        # 短暂休眠（让出CPU，可响应中断）
        time.sleep(interval)
```

**scroll.py 中断机制增强：**

```python

    # ★ 新增：检查运行状态（F12后is_running变为False）
    if not context.is_running:
        print(f"检测到停止信号，中断第{click_num+1}/{self.clicks}次点击")
        break  # 不再执行后续click
    
    context.execute_mouse_scroll(scroll_distance)

```

```
    ↓
```

**性能特性：**

- **多次滚动(clicks>1)**: F12后立即中断后续所有click
- **响应时间**: <0.1秒（取决于当前sleep间隔）
- **Normal版本兼容**: 同样受益于scroll.py的is\_running检查

**ScriptNode 执行器池管理：**

ScriptNode 使用类级别的执行器池管理脚本执行状态，确保节点重置时正确清理：

```python
class ScriptNode(ActionNode):
    _executor_pool: Dict[str, ScriptExecutor] = {}  # 类级别执行器池
    _pool_lock = threading.Lock()  # 线程安全锁
    
    def reset(self, reset_counters: bool = True) -> None:
        with self._lock:
            # 停止当前执行器
            if self._executor and self._executor.is_running:
                self._executor.stop_script()
            
            # 从执行器池中移除
            with ScriptNode._pool_lock:
                if self.node_id in ScriptNode._executor_pool:
                    del ScriptNode._executor_pool[self.node_id]
            
            self._executor = None
            self._script_started = False
            self._aborted = False
        super().reset(reset_counters=reset_counters)
```

**ScriptNode 循环执行逻辑：**

ScriptNode 的 `loop` 参数与 `repeat_count` 参数需要正确配合：

```python
# loop=true 且 repeat_count=0 时使用循环执行
use_loop = self.loop and self.config.repeat_count == 0
self._executor.run_script(self._script_content, loop=use_loop)
```

当 `repeat_count != 0` 时，由装饰器 `_execute_with_decorators()` 处理重复逻辑，不应同时启用 `loop`。

**ScriptExecutor delay 命令非阻塞实现：**

脚本中的 `delay` 命令需要支持中断，不能使用阻塞式 `time.sleep()`：

```python
elif command["type"] == "delay":
    delay_time = command["time"] / 1000
    elapsed = 0
    # 分段检查 is_running，支持中断
    while elapsed < delay_time and self.is_running:
        sleep_time = min(0.1, delay_time - elapsed)
        time.sleep(sleep_time)
        elapsed += sleep_time
```

这确保了用户点击停止时，脚本能够及时响应并停止执行。

```
            return ""
```

**逐字符输入机制：**

```
function _input_text(context, text):
    for char in text:
        if not context.check_running():
            break
        
        if char == '\n':
            context.execute_key_press("enter", "press", 0)
        elif char == '\t':
            context.execute_key_press("tab", "press", 0)
        else:
            pyperclip.copy(char)           // 复制字符到剪贴板
            context.execute_key_press("ctrl", "down", 0)
            context.execute_key_press("v", "press", 0)   // Ctrl+V 粘贴
            context.execute_key_press("ctrl", "up", 0)
        
        if input_delay > 0:
            sleep(input_delay / 1000.0)
```

**预设文本顺序执行逻辑：**

```
function _get_next_preset_text(context):
    index_key = node_id + "_text_index"
    current_index = blackboard.get(index_key, 0)
    
    text = preset_texts[current_index % len(preset_texts)]
    
    next_index = (current_index + 1) % len(preset_texts)
    blackboard.set(index_key, next_index)
    
    return text
```

**关键实现细节：**

- 继承 `ActionNode`，使用 `_execute_action()` 实现动作逻辑
- 使用 pyperclip + Ctrl+V 逐字符输入，兼容各种输入框（包括游戏内输入框）
- 换行符 `\n` 转换为 Enter 键，制表符 `\t` 转换为 Tab 键
- `input_delay` 控制字符输入间隔，避免输入过快导致丢失
- `clear_before_input` 使用 Ctrl+A 全选后输入新内容，实现清空效果
- 预设文本顺序执行使用黑板变量记录当前索引，支持循环使用
- `save_input_text` 将输入的文本保存到黑板，供后续节点使用
- `file_path` 已注册到 ResourceService，文件会自动导入到项目文件夹
- 序列化时使用中文显示名（`to_dict`），反序列化时通过映射表转换回内部值

### 2.1 日志管理器 (LogManager)

LogManager 是节点日志输出的核心组件，采用单例模式，提供统一的日志接口：

```python
class LogManager:
    _instance = None
    
    @classmethod
    def instance(cls) -> "LogManager":
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
    
    def log_success(self, node_type: str, node_name: str) -> None:
        """记录节点执行成功"""
        self._add_log("SUCCESS", node_type, node_name)
    
    def log_failure(self, node_type: str, node_name: str, reason: str = "") -> None:
        """记录节点执行失败"""
        message = f"{node_name}: 失败"
        if reason:
            message += f" - {reason}"
        self._add_log("FAILURE", node_type, message)
    
    def log_aborted(self, node_type: str, node_name: str) -> None:
        """记录节点被中止"""
        self._add_log("ABORTED", node_type, f"{node_name}: 已中止")
```

## 3 行为树序列化与反序列化

### 3.1 序列化方案

序列化采用"扁平化节点 + 连线列表"的结构，而非嵌套树结构，便于编辑器操作：

**序列化流程：**

```
Root Node (树结构)
    │
    │  collect_nodes() 递归遍历
    ▼
┌─────────────────────────────────────────┐
│ nodes: {                                │
│   "n1": { id, type, config, children }, │  ← 扁平化节点字典
│   "n2": { id, type, config, children }, │
│   "n3": { id, type, config, children }  │
│ }                                       │
│ connections: [                          │
│   { parent_id: "n1", child_id: "n2" }, │  ← 连线列表
│   { parent_id: "n1", child_id: "n3" }  │
│ ]                                       │
│ root_node: "n1"                         │  ← 根节点标识
└─────────────────────────────────────────┘
```

### 3.2 反序列化方案

**反序列化流程：**

```
JSON 数据
    │
    │ 1. 解析 nodes 字典
    ▼
┌──────────────────────────────────────┐
│ for each node_data in nodes:         │
│   node_class = NodeRegistry.get(type)│  ← 查找注册的节点类
│   node = node_class.from_dict(data)  │  ← 创建节点实例
│   nodes[node_id] = node              │
└──────────────────────────────────────┘
    │
    │ 2. 重建连接关系
    ▼
┌──────────────────────────────────────┐
│ for each conn in connections:        │
│   parent = nodes[conn.parent_id]     │
│   child = nodes[conn.child_id]       │
│   parent.add_child(child)            │  ← 重建父子关系
└──────────────────────────────────────┘
    │
    │ 3. 返回元组
    ▼
return (nodes[root_id], canvas_state, editor_state)
```

**返回值说明：**

`Serializer.deserialize()` 返回三元组：

| 元素            | 类型   | 说明                   |
| ------------- | ---- | -------------------- |
| root\_node    | Node | 反序列化的根节点             |
| canvas\_state | Dict | 画布状态（节点位置、视口偏移、缩放比例） |
| editor\_state | Dict | 编辑器状态（选中节点、文件路径）     |

**引擎与编辑器的不同处理：**

```python
# 引擎加载：仅需根节点
result = Serializer.deserialize(data)
if isinstance(result, tuple):
    root_node = result[0]
else:
    root_node = result  # 兼容旧格式
```

### 5.2 IbInputSimulator 实现细节

**DLL API：**

```
IbSendInit(send_type, mode, args)    // 初始化，send_type 选择驱动子模式
IbSendInputHook(mode)                // Hook 开关：1=ON, 0=OFF
IbSendDestroy()                      // 销毁，释放资源
```

**send_type 映射：**

| send_type | 驱动模式       | 说明                   |
| --------- | ---------- | -------------------- |
| 0         | AnyDriver  | 自动选择可用驱动             |
| 1         | SendInput  | Win32 SendInput，移除注入标志 |
| 2         | Logitech   | 罗技驱动模拟               |
| 3         | Razer      | 雷蛇驱动模拟               |
| 5         | MCII       | 内核级鼠标注入，需目标 PID      |
| 6         | GHubNew    | 罗技 G Hub 新版驱动        |

**Hook 机制：**

每次输入操作的标准流程：

```
IbSendInputHook(1)     // Hook ON
SendInput(...)          // 执行输入
IbSendInputHook(0)     // Hook OFF
```

Hook 拦截 SendInput 调用，移除 `injected` 标志，使输入表现为硬件输入。

**鼠标操作：**

```
// 定位：SetCursorPos + SendInput(MOVE|ABSOLUTE)
SetCursorPos(x, y)
SendInput(MOUSEINPUT: dx=0, dy=0, flags=MOUSEEVENTF_MOVE|MOUSEEVENTF_ABSOLUTE)

// 点击：SendInput(button flags)
左键按下: SendInput(MOUSEINPUT: flags=MOUSEEVENTF_LEFTDOWN)
左键抬起: SendInput(MOUSEINPUT: flags=MOUSEEVENTF_LEFTUP)
右键按下: SendInput(MOUSEINPUT: flags=MOUSEEVENTF_RIGHTDOWN)
右键抬起: SendInput(MOUSEINPUT: flags=MOUSEEVENTF_RIGHTUP)
```

**键盘操作：**

```
// 使用 VK 码直接发送
按下: SendInput(KEYBDINPUT: wVk=vk_code, wScan=0, flags=0)
抬起: SendInput(KEYBDINPUT: wVk=vk_code, wScan=0, flags=KEYEVENTF_KEYUP)
```

**MCII 模式：**

MCII 模式需要目标进程 PID，实现内核级鼠标注入：

```
IbSendInit(5, mode, target_pid)   // send_type=5, args=目标PID
```

**SendInput 模式：**

send_type=1 时，Hook 移除 SendInput 的 `injected` 标志，输入表现为硬件输入，可绕过部分应用层的注入检测。

| 操作     | 消息              | WPARAM                  | LPARAM                    |
| ------ | --------------- | ----------------------- | ------------------------- |
| 键盘按下   | WM_KEYDOWN      | vk_code                 | (scan_code << 16) \| 1    |

```
screen_x = world_x * zoom + pan_x
screen_y = world_y * zoom + pan_y

world_x = (screen_x - pan_x) / zoom
world_y = (screen_y - pan_y) / zoom
```

### 6.3.1 Canvas虚拟化（V1.5.2a 新增）

为支持数百节点流畅交互，画布新增虚拟化渲染机制，仅渲染视口内可见节点：

```python
class BehaviorTreeCanvas:
    def __init__(self):
        ...
        self._visible_node_ids: Set[str] = set()  # 当前视口内可见的节点ID
        self._virtualization_enabled = True         # 虚拟化开关

    def _update_visible_nodes(self):
        """核心虚拟化方法：根据视口动态增删Canvas图元"""
        new_visible = {nid for nid, node in self.nodes.items()
                       if self._is_node_visible(node)}

        # 离开视口的节点：删除Canvas图元
        for node_id in (self._visible_node_ids - new_visible):
            self.canvas.delete(node_id)

        # 持续可见的节点：更新zoom/pan并重绘
        for node_id in (self._visible_node_ids & new_visible):
            self.nodes[node_id].set_zoom(self.zoom)
            self.nodes[node_id].set_pan(self.pan_x, self.pan_y)
            self.nodes[node_id].redraw()

        # 新进入视口的节点：创建Canvas图元
        for node_id in (new_visible - self._visible_node_ids):
            self.nodes[node_id].set_zoom(self.zoom)
            self.nodes[node_id].set_pan(self.pan_x, self.pan_y)
            self.nodes[node_id].redraw()

        self._visible_node_ids = new_visible
        self._redraw_visible_connections()

    def _redraw_visible_connections(self):
        """仅绘制两端节点都在视口内的连线"""
        for parent_id, child_id in self.connections:
            if parent_id not in self._visible_node_ids or \
               child_id not in self._visible_node_ids:
                continue  # 跳过不可见连线
            self._draw_single_connection(parent_id, child_id, order_num)
```

**连线按需重绘（V1.5.2a 新增）：**

拖拽节点时仅重绘关联连线，使用 `_node_connections_map` 反向索引：

```python
def _redraw_connections_for_nodes(self, node_ids):
```

### 6.4 连接验证机制

为防止子节点被多个父节点连接导致的数据不一致问题，画布在创建连接时进行严格的验证。

#### 6.4.1 问题背景

行为树的节点模型设计如下：

```python
class Node:
    def __init__(self):
        self.children: List[Node] = []      # 可以有多个子节点
        self.parent: Optional[Node] = None   # 只能有一个父节点
```

**问题**：当一个子节点被多个父节点连接时：

1. `add_child()` 方法会直接覆盖子节点的 `parent` 引用
2. 但之前的父节点的 `children` 列表中仍然保留该子节点
3. 导致数据不一致：子节点的 `parent` 指向新父节点，但旧父节点仍认为它是自己的子节点

#### 6.4.2 验证实现

**add\_connection 方法实现：**

```python
def add_connection(self, parent_id: str, child_id: str) -> bool:
    """添加连接（带完整验证）"""
    from bt_utils.log_manager import LogManager
    
    # 1. 检查节点是否存在
    if parent_id not in self.nodes or child_id not in self.nodes:
        return False
    
    # 2. 检查是否已经存在相同的连接
    if any(c[0] == parent_id and c[1] == child_id for c in self.connections):
        return False
    
    # 3. 检查子节点是否已经有父节点
    existing_parents = [c[0] for c in self.connections if c[1] == child_id]
    if existing_parents:
        child_node = self.nodes.get(child_id)
        child_name = child_node.name if child_node else child_id
        LogManager.instance().log_info(
            "连接操作",
            child_name,
            "节点已有父节点，不能重复连接"
        )
        return False
    
    # 4. 检查是否会形成循环
    if self._would_create_cycle(parent_id, child_id):
        LogManager.instance().log_info(
            "连接操作",
            "循环检测",
            "不能形成循环连接"
        )
        return False
    
    # 5. 添加连接
    self.connections.append((parent_id, child_id))
    self._redraw_connections()
    return True
```

**循环检测算法：**

使用深度优先搜索（DFS）检测循环连接：

```python
def _would_create_cycle(self, parent_id: str, child_id: str) -> bool:
    """检查添加连接是否会形成循环"""
    visited = set()
    stack = [child_id]
    
    while stack:
        current = stack.pop()
        if current == parent_id:
            return True  # 找到循环
        if current in visited:
            continue
        visited.add(current)
        
        # 添加当前节点的所有子节点
        for p, c in self.connections:
            if p == current:
                stack.append(c)
    
    return False
```

#### 6.4.3 错误提示策略

连接失败时通过日志管理器输出提示信息，不使用弹窗干扰用户操作：

| 错误类型 | 日志输出示例                                 |
| ---- | -------------------------------------- |
| 多父节点 | `[时间] ℹ️ 连接操作 "动作节点" - 节点已有父节点，不能重复连接` |
| 循环连接 | `[时间] ℹ️ 连接操作 "循环检测" - 不能形成循环连接`       |

**设计优势：**

1. **非阻塞式提示**：不使用弹窗，避免打断用户操作流程
2. **信息清晰**：明确指出错误原因和涉及的节点
3. **统一入口**：所有日志集中显示在日志面板，便于查看历史记录

### 6.5 自动布局

当加载的树数据不包含位置信息时，使用递归自动布局：

```
function auto_layout(root_id):
    positions = {}
    
    function calc_subtree_width(node_id):
        children = get_children(node_id)
        if no children: return NODE_WIDTH (180)
        return max(sum(calc_subtree_width(c) for c in children), NODE_WIDTH)
    
    function layout_node(node_id, x, y):
        positions[node_id] = (x, y)
        children = get_children(node_id)
        if children:
            total_width = sum(calc_subtree_width(c) for c in children)
            current_x = x - total_width / 2
            for child in children:
                child_width = calc_subtree_width(child)
                layout_node(child, current_x + child_width/2, y + NODE_HEIGHT + V_GAP)
                current_x += child_width
    
    layout_node(root_id, 400, 100)
    return positions
```

### 8.6 智能放大策略 (v1.3.0 新增)

SmartScaleStrategy 计算安全的放大倍数，避免过度失真：

```python
class SmartScaleStrategy:
    """智能放大策略"""
    
    TARGET_CHAR_HEIGHT = 15     # 目标字符高度
    MAX_SCALE_FACTOR = 4.0      # 最大放大倍数
    
    def calculate_scale_factor(self, features: ImageFeatures) -> float:
        """计算安全的放大倍数"""
        if not features.is_small_font:
            return 1.0
        
        # 基于字符高度计算放大倍数
        char_based_scale = self.TARGET_CHAR_HEIGHT / features.estimated_char_height
        
        # 限制最大放大倍数
        return min(char_based_scale, self.MAX_SCALE_FACTOR)
    
    def scale_image(self, image: Image.Image, scale_factor: float) -> Image.Image:
        """使用Lanczos高质量插值放大图像"""
        if scale_factor <= 1.0:
            return image
        new_size = (int(image.size[0] * scale_factor), int(image.size[1] * scale_factor))
        return image.resize(new_size, Image.LANCZOS)
```

```python
def _normalize_key(self, key_name: str) -> str:

    parts = key_name.split("+")
    parts = [p.strip() for p in parts]

        else: normalized_parts.append(part)
    return "+".join(sorted(normalized_parts))
```

```python
def _calculate_paste_offset(self, relative_positions: Dict[str, tuple] = None) -> tuple:

    # 计算粘贴组边界框
    paste_width = max(rel_x for rel_x, rel_y in relative_positions.values()) + 160
    paste_height = max(rel_y for rel_x, rel_y in relative_positions.values()) + 70
    
    # 智能偏移避免重叠
    offset_x, offset_y = canvas_center_x, canvas_center_y
    for attempt in range(50):
        has_overlap = False
        for existing_node in self.canvas.nodes.values():
            # 矩形重叠检测
            if (offset_x < existing_node.x + 160 and 
                offset_x + paste_width > existing_node.x and
                offset_y < existing_node.y + 70 and 
                offset_y + paste_height > existing_node.y):
                has_overlap = True
                break
        
        if not has_overlap:
            break
        
        offset_x += 80
        offset_y += 80
    
    return offset_x, offset_y
```

**关键实现细节：**

| 要点     | 说明                                     |
| ------ | -------------------------------------- |
| 视口中心计算 | 使用 `(screen_center - pan) / zoom` 转换坐标 |
| 边界框计算  | 计算粘贴组的最大宽度和高度                          |
| 重叠检测   | 使用标准矩形重叠算法检测整个边界框                      |
| 偏移策略   | 每次偏移80像素，最多尝试50次                       |

### 12.1 PathResolver 路径解析器

PathResolver 负责相对路径与绝对路径之间的转换，确保项目的可移植性。

```python
class PathResolver:
    """资源路径解析器"""
    
    def __init__(self, project_root: str):
        self.project_root = os.path.abspath(project_root)
    
    def to_relative(self, absolute_path: str) -> str:
        """将绝对路径转换为相对路径"""
        abs_path = os.path.abspath(absolute_path)
        rel_path = os.path.relpath(abs_path, self.project_root)
        return f"./{rel_path.replace(os.sep, '/')}"
    
    def to_absolute(self, relative_path: str) -> str:
        """将相对路径转换为绝对路径"""
        if relative_path.startswith("./"):
            relative_path = relative_path[2:]
        
        abs_path = os.path.join(self.project_root, relative_path)
        return os.path.normpath(abs_path)
    
    def is_valid_relative_path(self, path: str) -> bool:
        """检查相对路径是否有效"""
        if not path.startswith("./"):
            return False
        
        abs_path = self.to_absolute(path)
        if not os.path.exists(abs_path):
            return False
        
        real_abs_path = os.path.realpath(abs_path)
        real_project_root = os.path.realpath(self.project_root)
        
        return real_abs_path.startswith(real_project_root)
```

**设计要点：**

- 使用 `os.path.relpath()` 计算相对路径
- 统一使用 `/` 作为路径分隔符（跨平台兼容）
- 路径有效性验证防止目录遍历攻击

```python
        if resource_type is None:
            resource_type = self._detect_resource_type(source_path)

        filename = os.path.basename(source_path)
        filename = self._handle_name_conflict(target_dir, filename)

        os.makedirs(os.path.dirname(target_path), exist_ok=True)
        shutil.copy2(source_path, target_path)
        
        return self.resolver.to_relative(target_path)
    
    def _detect_resource_type(self, path: str) -> str:
        """检测资源类型"""
        ext = os.path.splitext(path)[1].lower()

            return 'image'
        elif ext in ['.py', '.bat', '.sh', '.ps1']:
            return 'script'
        elif ext in ['.mp3', '.wav', '.ogg', '.flac']:
            return 'audio'
        elif ext in ['.json', '.yaml', '.xml', '.txt']:
            return 'data'
        else:
            return 'other'
```

### 12.5 PackageImporter 项目导入器

PackageImporter 从 ZIP 压缩包导入项目到默认工作区。

```python
class PackageImporter:
    """项目导入器"""
    
    REQUIRED_FILES = ["project.json", "tree.json"]
    
    def __init__(self):
        pass
    
    def validate_package(self, zip_path: str) -> Tuple[bool, str]:
        """验证 ZIP 文件是否为有效的项目压缩包
        
        Args:
            zip_path: ZIP 文件路径
            
        Returns:
            Tuple[bool, str]: (是否有效, 错误信息)
        """
        if not os.path.exists(zip_path):
            return False, "文件不存在"
        
        if not zipfile.is_zipfile(zip_path):
            return False, "不是有效的 ZIP 文件"
        
        try:
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                namelist = zipf.namelist()
                
                if not namelist:
                    return False, "ZIP 文件为空"
                
                found_project_json = False
                found_tree_json = False
                
                for name in namelist:
                    basename = os.path.basename(name)
                    if basename == "project.json":
                        found_project_json = True
                    elif basename == "tree.json":
                        found_tree_json = True
                
                if not found_project_json:
                    return False, "缺少 project.json 文件"
                
                if not found_tree_json:
                    return False, "缺少 tree.json 文件"
                
                return True, ""
                
        except zipfile.BadZipFile:
            return False, "ZIP 文件已损坏"
        except Exception as e:
            return False, f"读取 ZIP 文件失败: {str(e)}"
    
    def get_project_name(self, zip_path: str) -> Optional[str]:
        """从 ZIP 文件中获取项目名称"""
        try:
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                namelist = zipf.namelist()
                
                if not namelist:
                    return None
                
                # 从第一个条目获取项目文件夹名
                first_entry = namelist[0]
                if '/' in first_entry:
                    project_folder = first_entry.split('/')[0]
                    return project_folder
                
                # 尝试从 project.json 获取项目名
                for name in namelist:
                    basename = os.path.basename(name)
                    if basename == "project.json":
                        import json
                        with zipf.open(name) as f:
                            project_config = json.load(f)
                            project_info = project_config.get("project_info", {})
                            return project_info.get("name", None)
                
                return None
                
        except Exception:
            return None
    
    def import_from_zip(
        self, 
        zip_path: str, 
        target_dir: str,
        overwrite: bool = False,
        new_name: Optional[str] = None
    ) -> Tuple[bool, str, Optional[str]]:
        """从 ZIP 文件导入项目
        
        Args:
            zip_path: ZIP 文件路径
            target_dir: 目标目录
            overwrite: 是否覆盖已存在的项目
            new_name: 新项目名称（用于重命名）
            
        Returns:
            Tuple[bool, str, Optional[str]]: (是否成功, 错误信息, 导入后的项目路径)
        """
        is_valid, error_msg = self.validate_package(zip_path)
        if not is_valid:
            return False, error_msg, None
        
        project_name = new_name or self.get_project_name(zip_path)
        if not project_name:
            project_name = os.path.splitext(os.path.basename(zip_path))[0]
        
        project_root = os.path.join(target_dir, project_name)
        
        if os.path.exists(project_root) and not overwrite:
            return False, "PROJECT_EXISTS", project_root
        
        try:
            os.makedirs(target_dir, exist_ok=True)
            
            if os.path.exists(project_root):
                shutil.rmtree(project_root)
            
            with zipfile.ZipFile(zip_path, 'r') as zipf:
                zip_root = self.get_project_root_in_zip(zip_path)
                
                for member in zipf.namelist():
                    if zip_root:
                        if not member.startswith(zip_root + "/"):
                            continue
                        relative_path = member[len(zip_root) + 1:]
                    else:
                        relative_path = member
                    
                    if not relative_path or relative_path.endswith("/"):
                        continue
                    
                    target_path = os.path.join(project_root, relative_path)
                    os.makedirs(os.path.dirname(target_path), exist_ok=True)
                    
                    with zipf.open(member) as source, open(target_path, 'wb') as target:
                        target.write(source.read())
            
            return True, "", project_root
            
        except Exception as e:
            return False, f"解压失败: {str(e)}", None
    
    def generate_new_name(self, base_name: str, target_dir: str) -> str:
        """生成不冲突的新项目名称"""
        if not os.path.exists(os.path.join(target_dir, base_name)):
            return base_name
        
        counter = 1
        while True:
            new_name = f"{base_name}_{counter}"
            if not os.path.exists(os.path.join(target_dir, new_name)):
                return new_name
            counter += 1
```

- 检查文件是否存在
- 检查是否为有效的 ZIP 文件
- 检查 ZIP 内是否包含 `project.json`
- 检查 ZIP 内是否包含 `tree.json`

```python
    project_name = importer.get_project_name(zip_path)
    target_project_path = os.path.join(workspace_path, project_name)
    
    if os.path.exists(target_project_path):
        # 询问用户处理方式
        result = messagebox.askyesnocancel("项目已存在", "...")
        if result is None:
            return  # 取消
        elif result:
            overwrite = True  # 覆盖
        else:
            new_name = importer.generate_new_name(project_name, workspace_path)
            overwrite = False  # 重命名
    
    # 执行导入
    success, error_msg, imported_path = importer.import_from_zip(
        zip_path, workspace_path, overwrite=overwrite, new_name=new_name
    )
    
    if success:
        # 打开导入的项目
        self.open_project(imported_path)
        messagebox.showinfo("成功", f"项目导入成功！")
```

### 12.6 ResourceService 资源管理服务

ResourceService 提供统一的资源管理功能，包括收集外部资源、导入资源到项目、缓存管理等。

```python
class ResourceService:
    """资源管理服务"""
    
    RESOURCE_TYPE_MAP = {
        'template_path': 'image',
        'script_path': 'script',
        'code_path': 'code',
        'sound_path': 'audio',
    }
    
    TYPE_DIR_MAP = {
        'image': 'images/templates',
        'script': 'scripts/script',
        'code': 'scripts/code',
        'audio': 'audio/alarms',
        'data': 'data/config',
        'other': 'data/other',
    }
    
    CACHE_DIR = 'cache'
    
    def __init__(self, project_root: str):
        self.project_root = project_root
        self.resolver = PathResolver(project_root)
    
    def collect_external_resources(self, tree_data: Dict) -> List[Dict]:
        """收集行为树中引用的外部资源"""
        external_resources = []
        nodes = tree_data.get('nodes', {})
        
        for node_id, node_data in nodes.items():
            for path_key, resource_type in self.RESOURCE_TYPE_MAP.items():
                path = node_data.get('config', {}).get(path_key)
                if path and not path.startswith('./'):
                    external_resources.append({
                        'node_id': node_id,
                        'path_key': path_key,
                        'path': path,
                        'resource_type': resource_type
                    })
        
        return external_resources
    
    def import_resources_to_project(self, resources: List[Dict]) -> Dict[str, str]:
        """批量导入资源到项目目录"""
        path_mapping = {}

        for resource in resources:
            new_path = self.import_single_file_to_project(
                resource['path'],
                resource['resource_type']
            )
            if new_path:
                path_mapping[resource['node_id'] + '.' + resource['path_key']] = new_path

        return path_mapping

    def import_single_file_to_project(
        self,
        source_path: str,
        resource_type: str
    ) -> str:
        """导入单个文件到项目目录（V1.5.1 移除 old_path 参数，旧文件清理由 save_with_cleanup 统一处理）"""
        target_dir = self.TYPE_DIR_MAP.get(resource_type, 'data/other')
        filename = os.path.basename(source_path)
        target_path = os.path.join(self.project_root, target_dir, filename)
        
        # 处理文件名冲突
        if os.path.exists(target_path):
            name, ext = os.path.splitext(filename)
            counter = 2
            while os.path.exists(target_path):
                filename = f"{name}_{counter}{ext}"
                target_path = os.path.join(self.project_root, target_dir, filename)
                counter += 1
        
        # 复制文件
        os.makedirs(os.path.dirname(target_path), exist_ok=True)
        shutil.copy2(source_path, target_path)

        return self.resolver.to_relative(target_path)
    
    def move_to_cache(self, file_path: str) -> bool:
        """将文件移动到缓存目录"""
        if not file_path or file_path.startswith('./'):
            return False
        
        if not os.path.exists(file_path):
            return False
        
        cache_dir = os.path.join(self.project_root, self.CACHE_DIR)
        os.makedirs(cache_dir, exist_ok=True)
        
        filename = os.path.basename(file_path)
        name, ext = os.path.splitext(filename)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        cache_filename = f"{name}_{timestamp}{ext}"
        cache_path = os.path.join(cache_dir, cache_filename)
        
        shutil.move(file_path, cache_path)
        return True
    
    def update_tree_paths(self, tree_data: Dict, path_mapping: Dict[str, str]) -> Dict:
        """更新行为树中的路径引用"""
        nodes = tree_data.get('nodes', {})

        for key, new_path in path_mapping.items():
            node_id, path_key = key.rsplit('.', 1)
            if node_id in nodes:
                if 'config' not in nodes[node_id]:
                    nodes[node_id]['config'] = {}
                nodes[node_id]['config'][path_key] = new_path

        return tree_data

    def get_project_resource_dirs(self) -> List[str]:
        """从 TYPE_DIR_MAP 自动生成项目资源目录列表（V1.5.1）"""
        return list(self.TYPE_DIR_MAP.values())

    def save_with_cleanup(self, tree_data: Dict, project_root: str) -> Dict:
        """统一资源处理入口（V1.5.1 新增）

        在保存项目时统一执行：
        1. 收集外部资源并导入到项目目录
        2. 更新行为树中的路径引用
        3. 清理未被引用的旧资源文件（移动到缓存目录）
        """
        # 收集外部资源
        external_resources = self.collect_external_resources(tree_data)

        if external_resources:
            # 导入资源到项目目录
            path_mapping = self.import_resources_to_project(external_resources)
            # 更新行为树中的路径引用
            tree_data = self.update_tree_paths(tree_data, path_mapping)

        # 清理未被引用的资源文件
        self._cleanup_unreferenced_resources(tree_data, project_root)

        return tree_data

    def _cleanup_unreferenced_resources(self, tree_data: Dict, project_root: str):
        """清理未被行为树引用的资源文件，移动到缓存目录"""
        # 收集所有被引用的相对路径
        referenced_paths = set()
        nodes = tree_data.get('nodes', {})
        for node_id, node_data in nodes.items():
            for path_key in self.RESOURCE_TYPE_MAP:
                path = node_data.get('config', {}).get(path_key)
                if path:
                    referenced_paths.add(path)

        # 检查每个资源目录，将未被引用的文件移到缓存
        for resource_dir in self.get_project_resource_dirs():
            full_dir = os.path.join(project_root, resource_dir)
            if not os.path.exists(full_dir):
                continue
            for filename in os.listdir(full_dir):
                file_path = os.path.join(full_dir, filename)
                if os.path.isfile(file_path):
                    rel_path = self.resolver.to_relative(file_path)
                    if rel_path not in referenced_paths:
                        self.move_to_cache(file_path)

    def cleanup_cache(self, project_root: str, max_age_days: int = 7) -> int:
        """缓存定期清理（V1.5.1 新增）

        清理缓存目录中超过 max_age_days 天的文件。
        返回清理的文件数量。
        """
        cache_dir = os.path.join(project_root, self.CACHE_DIR)
        if not os.path.exists(cache_dir):
            return 0

        now = datetime.now()
        cleaned_count = 0

        for filename in os.listdir(cache_dir):
            file_path = os.path.join(cache_dir, filename)
            if os.path.isfile(file_path):
                file_mtime = datetime.fromtimestamp(os.path.getmtime(file_path))
                if (now - file_mtime).days > max_age_days:
                    os.remove(file_path)
                    cleaned_count += 1

        return cleaned_count

    def move_dir_to_cache(self, dir_path: str, project_root: str) -> bool:
        """目录级移动到缓存（V1.5.1 新增）

        将整个目录移动到缓存目录，用于 SubtreeField 等需要替换整个目录的场景。
        """
        if not os.path.isdir(dir_path):
            return False

        cache_dir = os.path.join(project_root, self.CACHE_DIR)
        os.makedirs(cache_dir, exist_ok=True)

        dir_name = os.path.basename(dir_path)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        cache_dirname = f"{dir_name}_{timestamp}"
        cache_path = os.path.join(cache_dir, cache_dirname)

        shutil.move(dir_path, cache_path)
        return True
```

## 14 UI 更新分发器实现（防止假卡死）

### 14.1 问题背景

tkinter 的 GUI 操作必须在主线程执行，但行为树引擎在独立线程中运行。如果直接从后台线程调用 tkinter 的 `update()` 或 `event_generate()`，可能导致：

1. **界面假卡死**：点击"开始运行"后，界面看起来卡住，节点状态不更新
2. **更新延迟**：需要鼠标点击才能触发界面更新
3. **再次运行异常**：手动停止后再次运行，界面节点高亮和状态灯显示异常

### 14.2 核心解决方案

采用**定时轮询 + 后台守护**混合模式：

```python
# bt_utils/ui_dispatcher.py

class UIUpdateDispatcher:
    def __init__(self):
        self._task_queue: Queue = Queue()  # 线程安全队列
        self._widget = None
        self._polling_active = False
        self.POLLING_INTERVAL_RUNNING = 10   # V1.5.2a：运行时10ms轮询
        self.POLLING_INTERVAL_IDLE = 100     # V1.5.2a：空闲时100ms轮询
        self._node_status_cache: Dict[str, tuple] = {}  # 节点状态缓存
    
    def dispatch_node_status(self, node_id: str, status: str, callback: Callable = None):
        """从后台线程调用，分发节点状态更新"""
        if node_id:
            self._node_status_cache[node_id] = ({"node_id": node_id, "status": status}, callback)
        
        task = UpdateTask(UpdateType.NODE_STATUS, {"node_id": node_id, "status": status}, callback)
        self._task_queue.put(task)  # 线程安全
        # 注意：不调用 after(0, ...)，完全依赖轮询机制
```

### 14.3 轮询机制实现

**关键：轮询必须在编辑器初始化时启动，而不是运行时启动。**

```python

    pass
```

### 14.4 轮询守护机制

当 `_widget` 为 None 时，使用后台线程继续尝试调度：

```python
# bt_utils/ui_dispatcher.py

def _poll(self):
    """主线程轮询"""
    if not self._polling_active:
        return
    
    widget = self._widget
    
    try:
        if widget is not None:
            pending = self._task_queue.qsize()
            if pending > 0:
                self._process_updates()
    except Exception:
        pass
    
    if self._polling_active:
        if self._widget is not None:
            # 正常情况：主线程调度下一次轮询
            self._widget.after(self._polling_interval_ms, self._poll)
        else:
            # 异常情况：后台线程守护，等待 _widget 可用
            threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()

def _poll_daemon(self):
    """后台线程轮询，用于在 _widget 为 None 时继续尝试"""
    if self._polling_active and self._widget is not None:
        # _widget 可用，切换回主线程轮询
        self._widget.after(0, self._poll)
    elif self._polling_active:
        # 继续等待
        threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()
```

### 14.5 防止假卡死的关键要点

| 问题                  | 错误做法                       | 正确做法                            |
| ------------------- | -------------------------- | ------------------------------- |
| 启动时假卡死              | 在 `_start_running()` 中启动轮询 | 在 `_init_ui_dispatcher()` 中启动轮询 |
| 手动停止后异常             | 停止时调用 `stop_polling()`     | **不要停止轮询**，让它持续运行               |
| 轮询永久停止              | `_widget` 为 None 时直接返回     | 使用后台线程守护，等待 `_widget` 可用        |
| `after(0, ...)` 不可靠 | 从后台线程调用 `after(0, ...)`    | 完全依赖定时轮询机制                      |

### 21.1 SubtreeNode 节点实现

SubtreeNode 是 CompositeNode 的子类，用于在运行时动态加载并执行外部行为树项目：

```python
class SubtreeNode(CompositeNode):
    NODE_TYPE = "SubtreeNode"

    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.subtree_path = config.get("subtree_path", "") if config else ""
        self.blackboard_mode = config.get("blackboard_mode", "inherit") if config else "inherit"
        self.namespace = config.get("namespace", "") if config else ""
        self.auto_reload = config.get("auto_reload", True) if config else True
        self._subtree_root = None           # 加载后的子树根节点
        self._subtree_context = None         # 子树执行上下文
        self._loaded_path = None            # 已加载的路径（用于变化检测）
        self._subtree_project_dir = None     # 子树项目目录

    def _resolve_path(self) -> str:
        """将相对路径解析为绝对路径"""
        import os
        path = self.subtree_path
        if not path:
            return ""
        if os.path.isabs(path):
            return path
        project_root = getattr(self, '_project_root', '')
        if project_root:
            return os.path.normpath(os.path.join(project_root, path))
        return path

    def _find_tree_file(self, project_dir: str) -> str:
        """在子树项目文件夹中查找 tree.json"""
        # 1. 尝试 project.json → main_tree
        # 2. 尝试 tree.json
        # 3. 尝试唯一的 .json 文件

    def _ensure_loaded(self):
        """首次 tick 时加载子树"""
        if self._subtree_root is not None and (not self.auto_reload or self._loaded_path == self._resolve_path()):
            return
        # 解析路径 → 查找文件 → 循环检测 → 序列化加载

    def _create_subtree_context(self, parent_context):
        """创建子树执行上下文，根据 blackboard_mode 选择隔离策略"""
        # inherit: 新上下文 + 共享黑板引用
        # isolated: 新上下文 + 独立 Blackboard()
        # namespaced: 新上下文 + NamespacedBlackboard(parent_bb, namespace)

    def tick(self, context):
        """执行子树：加载 → 创建上下文 → 压栈 → 执行子树根节点"""
        self._ensure_loaded()
        if self._subtree_root is None:
            return NodeStatus.FAILURE
        sub_ctx = self._create_subtree_context(context)
        context.push_subtree(self._resolve_path())
        status = self._subtree_root.tick(sub_ctx)
        if status != NodeStatus.RUNNING:
            context.pop_subtree()
        return status

    def abort(self, context):
        """中止子树执行"""
        if self._subtree_root:
            self._subtree_root.abort(
                self._subtree_context or context
            )

    def reset(self, reset_counters=True):
        """重置状态并传播到子树所有节点"""
        super().reset(reset_counters)
        if self._subtree_root:
            self._subtree_root.reset(reset_counters)

    def to_dict(self):
        """序列化为字典"""
        result = super().to_dict()
        result["config"]["subtree_path"] = self.subtree_path
        result["config"]["blackboard_mode"] = self.blackboard_mode
        result["config"]["namespace"] = self.namespace
        result["config"]["auto_reload"] = self.auto_reload
        return result
```

**关键设计要点：**

| 要点   | 说明                                                    |
| ---- | ----------------------------------------------------- |
| 延迟加载 | 首次 tick 时才加载子树，避免启动时开销                                |
| 路径解析 | 支持相对路径和绝对路径，相对路径基于 project\_root 解析                   |
| 文件查找 | 按 project.json→main\_tree / tree.json / 单一json 的优先级查找 |
| 循环检测 | 使用 CycleDetector 在加载时 DFS 检测循环引用                      |
| 黑板隔离 | 三种模式：inherit/isolated/namespaced                      |
| 状态传播 | reset() 和 abort() 自动传播到子树所有节点                         |

### 21.2 CycleDetector 循环检测实现

CycleDetector 使用深度优先搜索（DFS）算法检测子树引用中的循环依赖：

```python
class CycleDetector:
    @staticmethod
    def check(subtree_refs: List[Dict], entry_path: str = "") -> Tuple[bool, Optional[List[str]]]:
        """
        检查是否存在循环引用

        Args:
            subtree_refs: 子树引用列表 [{"node_id": "...", "path": "./subtrees/xxx"}]
            entry_path: 入口项目路径（用于归一化）

        Returns:
            (has_cycle: bool, cycle_path: Optional[List[str]])
            has_cycle=True 表示发现循环，cycle_path 为循环路径上的节点列表
        """

    @staticmethod
    def check_with_context(context: 'ExecutionContext') -> Tuple[bool, Optional[List[str]]]:
        """结合当前执行上下文的循环检测（使用上下文的调用栈）"""

    @staticmethod
    def build_reference_graph(subtree_refs: List[Dict], base_path: str) -> Dict[str, Set[str]]:
        """构建子树引用关系图 {project_dir: set(引用的目标目录)}"""

    @staticmethod
    def detect_cycle_in_graph(graph: Dict[str, Set[str]], start: str) -> Optional[List[str]]:
        """DFS 检测图中是否存在从 start 开始的循环"""

    @staticmethod
    def _extract_subtree_refs(tree_data: Dict) -> List[Dict]:
        """从序列化的树数据中提取所有 SubtreeNode 的引用信息"""

    @staticmethod
    def _normalize_path(path: str) -> str:
        """路径归一化：统一分隔符、解析绝对路径、处理大小写"""
```

**算法流程：**

```
输入: subtree_refs = [
    {"path": "./subtrees/A"},
    {"path": "./subtrees/B"}
]
base_path = "D:/Project/Main"

步骤 1: 归一化所有路径
  ./subtrees/A → D:/Project/Main/subtrees/A
  ./subtrees/B → D:/Project/Main/subtrees/B

步骤 2: 构建引用图
  需要递归读取 A 和 B 项目中的 SubtreeNode 引用

步骤 3: DFS 遍历检测循环
  visited = {}   # 访问状态: 0=未访问, 1=访问中, 2=已完成
  path_stack = [] # 当前访问路径

  def dfs(node):
      if node in visited:
          if visited[node] == 1:  # 正在访问中 → 发现循环
              return True
          return False             # 已完成 → 无循环
      visited[node] = 1
      path_stack.append(node)
      for neighbor in graph.get(node, []):
          if dfs(neighbor):
              return True
      path_stack.pop()
      visited[node] = 2
      return False
```

**时间复杂度：** O(V + E)，其中 V 是项目数，E 是引用关系数

### 21.3 NamespacedBlackboard 命名空间黑板实现

NamespacedBlackboard 是 Blackboard 的代理类，通过键名前缀实现数据隔离：

```python
class NamespacedBlackboard(Blackboard):
    """命名空间黑板代理

    通过在键名前添加命名空间前缀来实现数据隔离，
    所有读写操作都代理到父级 Blackboard。
    """

    def __init__(self, parent_blackboard: Blackboard, namespace: str = ""):
        self._parent = parent_blackboard
        self._namespace = namespace

    def _wrap_key(self, key: str) -> str:
        """为键添加命名空间前缀"""
        if self._namespace:
            return f"{self._namespace}.{key}"
        return key

    def get(self, key, default=None):
        return self._parent.get(self._wrap_key(key), default)

    def set(self, key, value):
        self._parent.set(self._wrap_key(key), value)

    def increment(self, key, amount=1):
        self._parent.increment(self._wrap_key(key), amount)

    def delete(self, key):
        self._parent.delete(self._wrap_key(key))

    def exists(self, key) -> bool:
        return self._parent.exists(self._wrap_key(key))

    def get_all_keys(self) -> List[str]:
        """仅返回当前命名空间的键"""
        prefix = f"{self._namespace}."
        all_keys = self._parent.get_all_keys()
        return [k[len(prefix):] for k in all_keys
                if k.startswith(prefix)]

    def subscribe(self, key, callback):
        self._parent.subscribe(self._wrap_key(key), callback)

    def unsubscribe(self, key, callback):
        self._parent.unsubscribe(self._wrap_key(key), callback)
```

**三种黑板的创建方式对比：**

| 模式         | 创建代码                                                               | 数据可见性 |
| ---------- | ------------------------------------------------------------------ | ----- |
| inherit    | `ExecutionContext(parent=context, blackboard=context.blackboard)`  | 完全共享  |
| isolated   | `ExecutionContext(blackboard=Blackboard())`                        | 完全隔离  |
| namespaced | `ExecutionContext(blackboard=NamespacedBlackboard(parent_bb, ns))` | 前缀隔离  |

### 21.4 MultiTreeManager 多树管理器实现

MultiTreeManager 管理多个独立的行为树实例，支持线程安全的并发操作：

```python
import threading

class MultiTreeManager:
    def __init__(self, shared_blackboard: Blackboard = None):
        self._trees: Dict[str, TreeInstance] = {}
        self._lock = threading.Lock()
        self.shared_blackboard = shared_blackboard

    def add_tree(self, name: str, root_node: Node,
                 context: ExecutionContext = None,
                 engine: BehaviorTreeEngine = None) -> TreeInstance:
        """添加一个新的行为树实例"""
        with self._lock:
            if name in self._trees:
                raise ValueError(f"Tree '{name}' already exists")
            instance = TreeInstance(
                name=name,
                engine=engine or BehaviorTreeEngine(),
                context=context or ExecutionContext(),
                blackboard=self.shared_blackboard or Blackboard(),
                status="idle"
            )
            self._trees[name] = instance
            return instance

    def remove_tree(self, name: str) -> bool:
        """移除指定名称的行为树"""
        with self._lock:
            if name in self._trees:
                tree = self._trees[name]
                if tree.status == "running":
                    tree.engine.stop()
                del self._trees[name]
                return True
            return False

    def start_tree(self, name: str) -> bool:
        """启动指定名称的行为树"""
        with self._lock:
            tree = self._trees.get(name)
            if tree and tree.status != "running":
                tree.engine.set_root(tree.context, tree._root_node)
                tree.engine.start()
                tree.status = "running"
                tree.start_time = time.time()
                return True
            return False

    def stop_tree(self, name: str) -> bool:
        """停止指定名称的行为树"""
        with self._lock:
            tree = self._trees.get(name)
            if tree and tree.status == "running":
                tree.engine.stop()
                tree.status = "stopped"
                return True
            return False

    def start_all(self) -> int:
        """启动所有行为树，返回成功启动的数量"""
        count = 0
        with self._lock:
            for name in self._trees:
                if self.start_tree(name):
                    count += 1
        return count

    def stop_all(self) -> int:
        """停止所有行为树，返回成功停止的数量"""
        count = 0
        with self._lock:
            for name in list(self._trees.keys()):
                if self.stop_tree(name):
                    count += 1
        return count

    def get_tree_status(self, name: str) -> Optional[Dict]:
        """获取指定行为树的状态"""
        with self._lock:
            tree = self._trees.get(name)
            if tree:
                return tree.to_dict()
            return None

    def get_all_status(self) -> List[Dict]:
        """获取所有行为树的状态列表"""
        with self._lock:
            return [t.to_dict() for t in self._trees.values()]
```

```python
                return
            tab_keys = list(self._trees.keys())
            closed_index = tab_keys.index(tab_id) if tab_id in tab_keys else -1
            if instance.status == "running":
                instance.engine.stop()
            del self._trees[tab_id]
            if self._active_tab_id == tab_id:
                tab_ids = list(self._trees.keys())
                if tab_ids:
                    if closed_index < len(tab_ids):
                        self._active_tab_id = tab_ids[closed_index]
                    else:
                        self._active_tab_id = tab_ids[-1]
                else:
                    self._active_tab_id = None
        if self.on_tab_removed:
            self.on_tab_removed(tab_id)

    def start_tab(self, tab_id: str) -> bool:
        if self.on_tab_start_request:
            return self.on_tab_start_request(tab_id)
        instance = self._trees.get(tab_id)
        if not instance or instance.status == "running":
            return False
        instance.engine.start(instance.context)
        self.update_tab_status(tab_id, True)
        return True

    def stop_tab(self, tab_id: str) -> bool:
        if self.on_tab_stop_request:
            return self.on_tab_stop_request(tab_id)
        instance = self._trees.get(tab_id)
        if not instance or instance.status != "running":
            return False
        instance.engine.stop()
        self.update_tab_status(tab_id, False)
        return True
```

```python
        return

        instance = self.tab_manager.get_tab(tab_id)

        self._on_tab_engine_status_change(tab_id, status, ns)

        self._on_node_status(node_id, status, tid)
    context.set_tab_manager(self.tab_manager, tab_id)
    instance.engine = engine
    instance.context = context
    engine.start(context)
    self.tab_manager.update_tab_status(tab_id, True)

        self._play_start_sound()

        self._play_stop_sound()

    return True
```

### 21.6 SubtreeFileWatcher 文件监控实现

```python
from dataclasses import dataclass, field
from typing import Dict, Callable, List, Optional
import os

@dataclass
class SubtreeFileWatcher:
    """子树文件变化监控器

    通过比较文件的修改时间(mtime)来检测文件是否发生变化，
    支持热重载场景。
    """

    _watched_files: Dict[str, float] = field(default_factory=dict)
    _file_mtimes: Dict[str, float] = field(default_factory=dict)
    _lock = None  # threading.Lock()

    def watch(self, file_path: str, node_id: str) -> None:
        """注册文件监控"""
        abs_path = os.path.abspath(file_path)
        if os.path.exists(abs_path):
            mtime = os.path.getmtime(abs_path)
            with self._lock:
                self._watched_files[node_id] = abs_path
                self._file_mtimes[abs_path] = mtime

    def unwatch(self, node_id: str) -> None:
        """取消文件监控"""
        with self._lock:
            if node_id in self._watched_files:
                del self._watched_files[node_id]

    def get_watching_nodes(self) -> List[str]:
        """获取正在监控的节点ID列表"""
        with self._lock:
            return list(self._watched_files.keys())

    def check_changes(self) -> List[Tuple[str, str]]:
        """检查所有监控文件是否有变化

        Returns:
            [(node_id, file_path), ...] 变化的文件列表
        """
        changed = []
        with self._lock:
            for node_id, file_path in self._watched_files.items():
                if os.path.exists(file_path):
                    current_mtime = os.path.getmtime(file_path)
                    if current_mtime != self._file_mtimes.get(file_path):
                        changed.append((node_id, file_path))
                        self._file_mtimes[file_path] = current_mtime
        return changed

    def notify_changes(self, callback: Callable[[str, str], None]) -> int:
        """通知变化的文件并调用回调

        Args:
            callback: 回调函数 (node_id, file_path) -> None

        Returns:
            变化的文件数量
        """
        changes = self.check_changes()
        for node_id, file_path in changes:
            callback(node_id, file_path)
        return len(changes)
```

```python
            min_y = min(monitor.y for monitor in monitors)
            max_x = max(monitor.x + monitor.width for monitor in monitors)
            max_y = max(monitor.y + monitor.height for monitor in monitors)
            return (min_x, min_y, max_x, max_y)
    except ImportError:
        pass
    
    # Windows API 备用方案
    try:
        import ctypes
        user32 = ctypes.windll.user32
        virtual_left = user32.GetSystemMetrics(76)   # SM_XVIRTUALSCREEN
        virtual_top = user32.GetSystemMetrics(77)    # SM_YVIRTUALSCREEN
        virtual_width = user32.GetSystemMetrics(78)  # SM_CXVIRTUALSCREEN
        virtual_height = user32.GetSystemMetrics(79) # SM_CYVIRTUALSCREEN
        return (virtual_left, virtual_top, 
                virtual_left + virtual_width, 
                virtual_top + virtual_height)
    except Exception:
        pass
    
    return (0, 0, 1920, 1080)
```

### 15.7 按键捕获字段 (KeyField)

KeyField 用于捕获用户按下的按键，支持再次点击取消。v1.4.1b 改用 pynput 全局键盘监听替代 Tkinter 事件绑定，解决符号键采集名称不兼容问题（详见 key_name_resolver.py）。

**实现要点：**

```python
class KeyField(FieldWidget):
    def __init__(self, master, label, key, on_change, **kwargs):
        self._listening = False
        self._pynput_listener = None
        self._timeout_id = None
        super().__init__(master, label, key, on_change, **kwargs)
    
    def _start_listening(self):
        if self._listening:
            self._stop_listening()
            return
        self._listening = True
        self.btn.configure(text="取消", fg_color=self._dark_colors['warning'])
        toplevel = self.winfo_toplevel()
        if hasattr(toplevel, 'set_keyfield_active'):
            toplevel.set_keyfield_active(True)
        from pynput import keyboard
        from bt_utils.key_name_resolver import resolve_key_name
        def on_press(key):
            key_name = resolve_key_name(key)
            if key_name:
                self.after(0, lambda: self._on_key_captured(key_name))
                return False
        self._pynput_listener = keyboard.Listener(on_press=on_press)
        self._pynput_listener.start()
        self._timeout_id = self.after(10000, self._stop_listening)
    
    def _on_key_captured(self, key_name):
        if not self._listening:
            return
        self.var.set(key_name)
        self.on_change(self.key, key_name)
        self._stop_listening()
    
    def _stop_listening(self):
        self._listening = False
        if self._pynput_listener:
            self._pynput_listener.stop()
            self._pynput_listener = None
        if self._timeout_id:
            self.after_cancel(self._timeout_id)
            self._timeout_id = None
        toplevel = self.winfo_toplevel()
        if hasattr(toplevel, 'set_keyfield_active'):
            toplevel.set_keyfield_active(False)
        if self.winfo_exists():
            self.btn.configure(text="修改", fg_color=self._dark_colors['primary'])
```

**Delete/Backspace 拦截机制：**

pynput 和 Tkinter 是两套独立的事件系统。KeyField 监听时按 Delete/Backspace，pynput 捕获按键的同时，Tkinter 的 Delete/Backspace 快捷键仍会触发节点删除。通过 `_keyfield_active` 标记在 app.py 的 `_handle_shortcut` 中拦截。

```python
            self.frame,
            text="测量",
            width=50,
            command=self._measure_offset
        )
        self.measure_btn.pack(side="left", padx=5)
    
    def _measure_offset(self):
        """可视化偏移测量"""
        magnifier = MagnifierWindow(zoom_factor=4, size=150)
        magnifier_shown = [False]
        reference_point = [None]
        
        def on_mouse_move(event):
            if not magnifier_shown[0]:
                magnifier.show(event.x_root, event.y_root)
                magnifier_shown[0] = True
            else:
                magnifier.update(event.x_root, event.y_root)
        
        def on_click(event):
            if reference_point[0] is None:
                # 第一步：记录参考点
                reference_point[0] = (event.x_root, event.y_root)
            else:
                # 第二步：计算偏移
                target_point = (event.x_root, event.y_root)
                offset_x = target_point[0] - reference_point[0][0]
                offset_y = target_point[1] - reference_point[0][1]
                
                # 更新值
                self.entry.delete(0, "end")
                self.entry.insert(0, f"{offset_x}, {offset_y}")
                
                # 清理
                magnifier.hide()
                self._unbind_events()
        
        # 绑定事件
        self._bind_events(on_mouse_move, on_click)
```

## 20 窗口绑定功能实现

### 20.1 WindowManager 窗口管理器

WindowManager 提供窗口枚举、查找、切换等功能，使用 ctypes 调用 Windows API：

```python
# bt_utils/window_manager.py

import ctypes
from ctypes import wintypes
from typing import List, Tuple, Optional

class WindowManager:
    """窗口管理器，提供窗口枚举、查找、切换等功能"""
    
    @staticmethod
    def enum_all_windows() -> List[Tuple[int, str]]:
        """枚举所有可见窗口，返回窗口句柄和标题的列表"""
        user32 = ctypes.windll.user32
        
        windows = []
        
        def enum_windows_proc(hwnd, lparam):
            if user32.IsWindowVisible(hwnd):
                length = user32.GetWindowTextLengthW(hwnd)
                if length > 0:
                    buffer = ctypes.create_unicode_buffer(length + 1)
                    user32.GetWindowTextW(hwnd, buffer, length + 1)
                    title = buffer.value
                    if title:
                        windows.append((hwnd, title))
            return True
        
        WNDENUMPROC = ctypes.WINFUNCTYPE(wintypes.BOOL, wintypes.HWND, wintypes.LPARAM)
        user32.EnumWindows(WNDENUMPROC(enum_windows_proc), 0)
        
        return windows
    
    @staticmethod
    def find_window_by_title(keyword: str) -> Optional[int]:
        """根据标题关键字查找窗口（不区分大小写）"""
        windows = WindowManager.enum_all_windows()
        keyword_lower = keyword.lower()
        
        for hwnd, title in windows:
            if keyword_lower in title.lower():
                return hwnd
        
        return None
    
    @staticmethod
    def get_window_rect(hwnd: int) -> Optional[Tuple[int, int, int, int]]:
        """获取窗口矩形区域 (left, top, right, bottom)"""
        user32 = ctypes.windll.user32
        rect = wintypes.RECT()
        
        if user32.GetWindowRect(hwnd, ctypes.byref(rect)):
            return (rect.left, rect.top, rect.right, rect.bottom)
        
        return None
    
    @staticmethod
    def switch_to_window(hwnd: int) -> bool:
        """切换到指定窗口（置顶并激活）"""
        user32 = ctypes.windll.user32
        
        # 先恢复窗口（如果最小化）
        SW_RESTORE = 9
        user32.ShowWindow(hwnd, SW_RESTORE)
        
        # 设置窗口位置和大小
        SWP_NOMOVE = 0x0002
        SWP_NOSIZE = 0x0001
        SWP_SHOWWINDOW = 0x0040
        user32.SetWindowPos(hwnd, -1, 0, 0, 0, 0, SWP_NOMOVE | SWP_NOSIZE | SWP_SHOWWINDOW)
        
        # 设置前台窗口
        return user32.SetForegroundWindow(hwnd) != 0
    
    @staticmethod
    def restore_window(hwnd: int) -> bool:
        """恢复窗口（取消置顶）"""
        user32 = ctypes.windll.user32
        
        SWP_NOMOVE = 0x0002
        SWP_NOSIZE = 0x0001
        SWP_SHOWWINDOW = 0x0040
        SWP_NOZORDER = 0x0004
        
        return user32.SetWindowPos(hwnd, 0, 0, 0, 0, 0, 
                                   SWP_NOMOVE | SWP_NOSIZE | SWP_SHOWWINDOW | SWP_NOZORDER) != 0
    
    @staticmethod
    def get_foreground_window() -> Optional[int]:
        """获取当前前台窗口句柄"""
        user32 = ctypes.windll.user32
        return user32.GetForegroundWindow()
    
    @staticmethod
    def is_foreground_window(hwnd: int) -> bool:
        """检查指定窗口是否为前台窗口"""
        return WindowManager.get_foreground_window() == hwnd
    
    @staticmethod
    def find_window_by_pid(pid: int) -> Optional[int]:
        """通过进程ID查找窗口"""
        results = []
        
        WNDENUMPROC = ctypes.WINFUNCTYPE(wintypes.BOOL, wintypes.HWND, wintypes.LPARAM)
        
        def enum_windows_callback(hwnd, _):
            if user32.IsWindowVisible(hwnd):
                window_pid = wintypes.DWORD()
                user32.GetWindowThreadProcessId(hwnd, ctypes.byref(window_pid))
                if window_pid.value == pid:
                    length = user32.GetWindowTextLengthW(hwnd)
                    if length > 0:
                        buffer = ctypes.create_unicode_buffer(length + 1)
                        user32.GetWindowTextW(hwnd, buffer, length + 1)
                        if buffer.value:
                            results.append(hwnd)
            return True
        
        user32.EnumWindows(WNDENUMPROC(enum_windows_callback), 0)
        
        if results:
            return results[0]
        return None
    
    @staticmethod
    def get_window_pid(hwnd: int) -> Optional[int]:
        """获取窗口的进程ID"""
        try:
            pid = wintypes.DWORD()
            user32.GetWindowThreadProcessId(hwnd, ctypes.byref(pid))
            return pid.value if pid.value else None
        except Exception:
            return None
    
    @staticmethod
    def find_main_window_by_pid(pid: int) -> Optional[int]:
        """通过进程ID智能查找主窗口（多窗口时按优先级排序选择）"""
        WNDENUMPROC = ctypes.WINFUNCTYPE(wintypes.BOOL, wintypes.HWND, wintypes.LPARAM)
        results = []

        def enum_windows_callback(hwnd, _):
            if user32.IsWindowVisible(hwnd):
                window_pid = wintypes.DWORD()
                user32.GetWindowThreadProcessId(hwnd, ctypes.byref(window_pid))
                if window_pid.value == pid:
                    length = user32.GetWindowTextLengthW(hwnd)
                    if length > 0:
                        buffer = ctypes.create_unicode_buffer(length + 1)
                        user32.GetWindowTextW(hwnd, buffer, length + 1)
                        if buffer.value:
                            style = user32.GetWindowLongW(hwnd, -16)
                            ex_style = user32.GetWindowLongW(hwnd, -20)
                            WS_OVERLAPPEDWINDOW = 0x00CF0000
                            WS_EX_TOOLWINDOW = 0x00000080
                            WS_EX_APPWINDOW = 0x00040000
                            is_main = bool(style & WS_OVERLAPPEDWINDOW)
                            is_app_window = bool(ex_style & WS_EX_APPWINDOW)
                            is_tool_window = bool(ex_style & WS_EX_TOOLWINDOW)
                            rect = wintypes.RECT()
                            user32.GetClientRect(hwnd, ctypes.byref(rect))
                            area = max(0, (rect.right - rect.left)) * max(0, (rect.bottom - rect.top))
                            results.append({
                                'hwnd': hwnd, 'area': area,
                                'is_main': is_main, 'is_app_window': is_app_window,
                                'is_tool_window': is_tool_window, 'title': buffer.value,
                            })
            return True

        user32.EnumWindows(WNDENUMPROC(enum_windows_callback), 0)
        if not results:
            return None
        results.sort(key=lambda x: (
            not x['is_tool_window'], x['is_app_window'], x['is_main'], x['area']
        ), reverse=True)
        return results[0]['hwnd']

    @staticmethod
    def find_all_windows_by_pid(pid: int) -> List[int]:
        """通过进程ID查找所有可见窗口"""
        WNDENUMPROC = ctypes.WINFUNCTYPE(wintypes.BOOL, wintypes.HWND, wintypes.LPARAM)
        results = []

        def enum_windows_callback(hwnd, _):
            if user32.IsWindowVisible(hwnd):
                window_pid = wintypes.DWORD()
                user32.GetWindowThreadProcessId(hwnd, ctypes.byref(window_pid))
                if window_pid.value == pid:
                    length = user32.GetWindowTextLengthW(hwnd)
                    if length > 0:
                        buffer = ctypes.create_unicode_buffer(length + 1)
                        user32.GetWindowTextW(hwnd, buffer, length + 1)
                        if buffer.value:
                            results.append(hwnd)
            return True

        user32.EnumWindows(WNDENUMPROC(enum_windows_callback), 0)
        return results

    @staticmethod
    def find_window_smart(pid: Optional[int], title_keyword: str) -> Tuple[Optional[int], str]:
        """智能查找窗口：优先PID智能选择主窗口，其次关键字"""
        if pid:
            hwnd = WindowManager.find_main_window_by_pid(pid)
            if hwnd:
                return hwnd, "pid"
        if title_keyword:
            hwnd = WindowManager.find_window_by_title(title_keyword)
            if hwnd:
                return hwnd, "title"
        return None, "not_found"

    @staticmethod
    def find_window_smart_with_hwnd(hwnd: int, pid: Optional[int], title_keyword: str) -> Tuple[Optional[int], str]:
        """智能查找窗口：句柄>PID智能>标题，三级降级"""
        if hwnd and WindowManager.is_window_valid(hwnd):
            return hwnd, "hwnd"
        if pid:
            hwnd_found = WindowManager.find_main_window_by_pid(pid)
            if hwnd_found:
                return hwnd_found, "pid"
        if title_keyword:
            hwnd_found = WindowManager.find_window_by_title(title_keyword)
            if hwnd_found:
                return hwnd_found, "title"
        return None, "not_found"
```

```python
    def __init__(self, ...):

                result = WindowCapture.capture_window(self._bound_window)
        else:
            # 原有截图逻辑
            result = ...

        # 缓存结果
        if result is not None:
            self._screenshot_cache[cache_key] = result
        return result
    
    def convert_to_screen_coords(self, region: tuple) -> tuple:
        """将窗口相对坐标转换为屏幕绝对坐标"""
        if self._bound_window:
            from bt_utils.coordinate import CoordinateConverter
            return CoordinateConverter.window_region_to_screen(region, self._bound_window)
        return region
    
    def convert_to_window_coords(self, region: tuple) -> tuple:
        """将屏幕绝对坐标转换为窗口相对坐标"""
        if self._bound_window:
            from bt_utils.coordinate import CoordinateConverter
            return CoordinateConverter.screen_region_to_window(region, self._bound_window)
        return region
    
    def smart_switch_to_bound_window(self) -> bool:
        """智能切换到绑定窗口"""
        if not self._bound_window:
            return False
        
        from bt_utils.window_manager import WindowManager
        
        if WindowManager.is_foreground_window(self._bound_window):
            return True
        
        self._saved_foreground_window = WindowManager.get_foreground_window()
        return WindowManager.switch_to_window(self._bound_window)
    
    def smart_restore_foreground_window(self) -> bool:
        """智能恢复前台窗口"""
        if not self._bound_window or not self._saved_foreground_window:
            return False
        
        from bt_utils.window_manager import WindowManager
        
        if self._saved_foreground_window != self._bound_window:
            return WindowManager.switch_to_window(self._saved_foreground_window)
        
        return True
```

```python
class ActionNode(Node):
```

### 20.9 Editor.get\_start\_node 方法

Editor 提供了 `get_start_node` 方法用于获取 StartNode 实例，支持按节点类型查找而非硬编码 ID：

```python
# bt_gui/bt_editor/editor.py

class Editor:
    def get_start_node(self):
        """获取 StartNode 实例（按节点类型查找，而非硬编码 ID）"""
        for node_id, node in self.canvas.nodes.items():
            if node.node_type == "StartNode":
                from bt_core.config import NodeConfig
                config = NodeConfig(name=getattr(node, 'name', ''))
                node_config = node.config if hasattr(node, 'config') else {}
                for key, value in node_config.items():
                    config.set(key, value)
                from bt_core.nodes import StartNode
                return StartNode(node_id=node_id, config=config)
        return None
```

**设计说明：**

- 使用 `node_type == "StartNode"` 查找，而非硬编码 ID
- 兼容不同项目中可能存在的不同节点 ID 命名规则（如 `node_start`、`node_1` 等）
- 返回完整的 StartNode 实例，包含所有配置信息
