# 顶部Tab页签多行为树并行 — 前端设计方案

> 版本：v1.0
> 日期：2026-05-02
> 状态：待确认

---

## 1 设计理念

将当前的单画布编辑器改为**多Tab页签编辑器**，每个Tab对应一个独立的行为树项目（拥有独立的画布、引擎、上下文）。多个Tab可同时运行，实现多行为树并行。

**核心原则：**
- 每个Tab = 一个独立行为树项目
- Tab栏内置单树运行/停止控制
- 工具栏保留全局总控（全部运行/全部停止）
- 共享组件（节点面板、属性面板、日志面板）跟随当前活动Tab

## 3 Tab栏详细设计

### 3.1 Tab标签结构

每个Tab标签由三部分组成：

```
┌──────────────────────────┐
│  ▶  │  主任务      │  ✕  │
│运行 │  项目名称    │关闭  │
│按钮 │              │按钮  │
└──────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| **▶/⏹ 运行/停止按钮** | 切换按钮，控制该Tab对应行为树的运行/停止。运行中显示⏹（绿色），未运行显示▶（灰色） |
| **项目名称** | 显示项目文件夹名称，如"主任务"、"自动拾取"。当前活动Tab加粗高亮 |
| **✕ 关闭按钮** | 关闭该Tab（运行中需确认，未保存需确认） |

### 3.2 Tab栏右侧

| 元素 | 说明 |
|------|------|
| **[+导入]** | 点击后弹出文件夹选择对话框，导入新的行为树项目 |

### 3.3 Tab标签状态

| 状态 | 运行按钮 | 项目名称样式 | 标签背景 |
|------|----------|-------------|----------|
| **未运行 + 非活动** | ▶（灰色） | 正常 | 暗色 |
| **未运行 + 活动** | ▶（灰色） | 加粗 | 稍亮 |
| **运行中 + 非活动** | ⏹（绿色） | 正常 + 绿色圆点 | 暗色 |
| **运行中 + 活动** | ⏹（绿色） | 加粗 + 绿色圆点 | 稍亮 |

### 3.4 Tab标签交互

| 操作 | 行为 |
|------|------|
| **点击Tab标签** | 切换到该Tab（画布、属性面板切换） |
| **点击▶按钮** | 启动该Tab的行为树（不影响其他Tab） |
| **点击⏹按钮** | 停止该Tab的行为树 |
| **点击✕按钮** | 关闭该Tab（运行中提示停止，未保存提示保存） |
| **点击+导入** | 选择项目文件夹 → 复制到subtrees/ → 创建新Tab |

## 4 工具栏运行控制

### 4.1 现有控制保持不变

工具栏上方的运行控制保持现有样式，作为**全局总控**：

```
现有按钮（保持不变）：
  [▶ 运行]  —— 运行当前活动Tab的行为树
  [⏹ 停止]  —— 停止当前活动Tab的行为树
```

### 4.2 新增全局控制

在工具栏右侧新增两个全局控制按钮：

```
新增按钮：
  [▶▶ 全部运行]  —— 同时启动所有Tab的行为树
  [⏹⏹ 全部停止]  —— 同时停止所有Tab的行为树
```

### 4.3 运行控制层级

```
┌─────────────────────────────────────────────────────┐
│  工具栏总控                                           │
│  [▶ 运行]     → 控制当前活动Tab                       │
│  [⏹ 停止]     → 控制当前活动Tab                       │
│  [▶▶ 全部运行] → 控制所有Tab                          │
│  [⏹⏹ 全部停止] → 控制所有Tab                          │
├─────────────────────────────────────────────────────┤
│  Tab栏单控                                           │
│  [▶/⏹ 按钮]   → 仅控制该Tab                          │
│  可在任何Tab上操作，无需切换到该Tab即可控制              │
└─────────────────────────────────────────────────────┘
```

**关键特性：** Tab栏上的▶/⏹按钮允许在查看其他Tab时控制任意Tab的运行/停止，无需切换到目标Tab。

## 5 核心组件设计

### 5.1 TabContent 数据类

```python
@dataclass
class TabContent:
    """标签页内容"""
    tab_id: str                          # 唯一标识
    name: str                            # 显示名称（项目文件夹名）
    project_root: str                    # 项目根目录
    canvas: BehaviorTreeCanvas           # 画布实例
    engine: Optional[BehaviorTreeEngine] # 运行引擎
    context: Optional[ExecutionContext]  # 执行上下文
    file_path: Optional[str]             # 文件路径
    modified: bool = False               # 是否修改
    is_running: bool = False             # 是否运行中
```

### 5.2 TabBar 组件

```python
class TabBar(ctk.CTkFrame):
    """顶部标签栏"""

    tabs: Dict[str, TabButton]           # {tab_id: TabButton}
    active_tab_id: str                   # 当前活动Tab
    on_tab_switch: Callable              # Tab切换回调
    on_tab_close: Callable               # Tab关闭回调
    on_tab_run: Callable                 # Tab运行回调
    on_tab_stop: Callable                # Tab停止回调
    on_import: Callable                  # 导入项目回调
```

### 5.3 TabButton 组件

```python
class TabButton(ctk.CTkFrame):
    """单个Tab标签按钮"""

    tab_id: str
    name: str
    is_running: bool

    # UI元素
    run_stop_btn: ctk.CTkButton         # ▶/⏹ 运行/停止切换按钮
    name_label: ctk.CTkLabel            # 项目名称
    close_btn: ctk.CTkButton            # ✕ 关闭按钮
    status_indicator: ctk.CTkLabel      # 运行状态指示（绿色圆点）

    def set_running(running: bool):
        """更新运行状态显示"""
        # running=True: 按钮变为⏹(绿色), 显示绿色圆点
        # running=False: 按钮变为▶(灰色), 隐藏绿色圆点

    def set_active(active: bool):
        """更新活动状态显示"""
        # active=True: 名称加粗, 背景稍亮
        # active=False: 名称正常, 背景暗色
```

### 5.4 TabManager 管理器

```python
class TabManager:
    """标签页管理器"""

    tabs: Dict[str, TabContent]          # {tab_id: TabContent}
    active_tab_id: str                   # 当前活动Tab
    tab_bar: TabBar                      # Tab栏UI
    canvas_container: ctk.CTkFrame       # 画布容器

    def add_tab(project_path: str) -> str:
        """导入项目创建新Tab"""

    def remove_tab(tab_id: str) -> bool:
        """关闭Tab"""

    def switch_tab(tab_id: str) -> None:
        """切换活动Tab（隐藏旧画布，显示新画布，刷新属性面板）"""

    def get_active_tab() -> TabContent:
        """获取当前活动Tab"""

    def start_tab(tab_id: str) -> None:
        """启动指定Tab的行为树"""

    def stop_tab(tab_id: str) -> None:
        """停止指定Tab的行为树"""

    def start_all() -> None:
        """启动所有Tab的行为树"""

    def stop_all() -> None:
        """停止所有Tab的行为树"""

    def update_tab_status(tab_id: str, running: bool) -> None:
        """更新Tab运行状态（Tab栏按钮+指示器）"""
```

## 6 交互流程

### 6.1 导入新项目

```
1. 点击 Tab栏 [+导入] 按钮
2. 弹出文件夹选择对话框
3. 选择行为树项目文件夹（包含 project.json / tree.json）
4. 系统将项目文件夹复制到当前工作区 subtrees/ 目录
5. 创建新 TabContent（独立画布、引擎、上下文）
6. 在 Tab栏 添加新标签
7. 自动切换到新 Tab
```

### 6.2 运行单个行为树（方式1：Tab栏控制）

```
1. 在任意Tab上点击 ▶ 按钮
2. 该Tab的行为树启动运行
3. ▶ 按钮变为 ⏹（绿色），显示绿色圆点
4. 无需切换到该Tab即可操作
```

### 6.3 运行单个行为树（方式2：工具栏控制）

```
1. 切换到目标Tab（点击Tab标签）
2. 点击工具栏 [▶ 运行] 按钮
3. 当前活动Tab的行为树启动运行
```

### 6.4 多行为树并行运行

```
方式1：逐个启动
  - 在Tab A 上点击 ▶ → Tab A 运行
  - 在Tab B 上点击 ▶ → Tab B 运行
  - 两个行为树同时运行

方式2：一键全部启动
  - 点击工具栏 [▶▶ 全部运行] 按钮
  - 所有Tab的行为树同时启动
  - 所有Tab标签显示 ⏹ + 绿色圆点
```

### 6.5 停止运行

```
单树停止：
  - 在运行中的Tab上点击 ⏹ 按钮
  - 或切换到该Tab后点击工具栏 [⏹ 停止]

全部停止：
  - 点击工具栏 [⏹⏹ 全部停止] 按钮
  - 所有运行中的行为树同时停止
```

### 6.6 关闭Tab

```
1. 点击Tab上的 ✕ 按钮
2. 检查：行为树是否运行中？
   - 是 → 弹窗提示"行为树正在运行，是否停止并关闭？"
   - 否 → 继续
3. 检查：是否有未保存修改？
   - 是 → 弹窗提示"是否保存修改？"
   - 否 → 继续
4. 销毁画布、停止引擎、移除Tab
5. 切换到前一个Tab（或第一个Tab）
```

### 6.7 切换Tab

```
1. 点击Tab标签（非▶/⏹/✕区域）
2. 隐藏当前活动Tab的画布
3. 显示目标Tab的画布
4. 刷新属性面板（显示目标Tab的选中节点属性）
5. 更新工具栏运行按钮状态（反映当前Tab的运行状态）
```

## 8 与现有架构的兼容性

| 现有组件 | 改动类型 | 改动说明 |
|----------|----------|----------|
| `BehaviorTreeEditor` | **重构** | 从单画布管理改为多Tab管理，核心逻辑提取到TabContent |
| `BehaviorTreeCanvas` | **无改动** | 每个Tab创建独立实例 |
| `EditorToolbar` | **小改** | 新增"全部运行/全部停止"按钮 |
| `PropertyPanel` | **无改动** | 切换Tab时刷新显示 |
| `LogPanel` | **小改** | 日志前缀添加Tab名称 |
| `PalettePanel` | **无改动** | 拖拽目标切换为当前活动画布 |
| `CommandManager` | **小改** | 每个Tab独立撤销/重做栈 |

## 9 实现步骤

```
阶段1：TabContent + TabManager 数据层
  ├── TabContent 数据类
  ├── TabManager 核心逻辑（add/remove/switch）
  └── 单元测试

阶段2：TabBar UI 组件
  ├── TabButton 组件（▶/⏹按钮 + 名称 + ✕关闭）
  ├── TabBar 容器（标签列表 + [+导入]按钮）
  └── 状态更新（运行指示器）

阶段3：Editor 集成
  ├── 替换单画布为多Tab画布容器
  ├── Tab切换时画布显示/隐藏
  ├── 属性面板跟随Tab切换
  └── 撤销/重做栈按Tab隔离

阶段4：运行控制
  ├── Tab栏单树运行/停止
  ├── 工具栏当前Tab运行/停止（保持现有逻辑）
  ├── 工具栏全部运行/全部停止
  └── 运行状态同步（Tab栏 ↔ 工具栏）

阶段5：导入项目
  ├── 文件夹选择 → 复制到 subtrees/
  ├── 创建新Tab → 加载项目
  └── 自动切换到新Tab

阶段6：关闭与保存
  ├── 关闭Tab前检查（运行状态 + 修改状态）
  ├── 保存当前Tab
  └── 保存所有Tab
```

## 10 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 多引擎并行线程安全 | 数据竞争 | 每个引擎独立线程，共享资源加锁 |
| 画布切换闪烁 | 用户体验差 | 预创建画布，show/hide而非destroy/create |
| 内存占用增加 | 多画布多引擎 | 限制最大Tab数量（如10个） |
| 日志混淆 | 难以区分来源 | 日志前缀添加Tab名称 |
| 撤销/重做跨Tab | 操作混乱 | 每个Tab独立撤销栈 |
