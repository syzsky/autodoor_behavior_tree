# AutoDoor 行为树系统 — 架构文档

## 1 项目概述

### 1.1 项目定位

AutoDoor 行为树系统是一个独立的可视化行为树编辑与执行框架，面向 Windows 平台的自动化场景（客户端辅助、RPA 流程等）。系统提供完整的图形化编辑器、多种检测/动作节点、脚本录制、OCR 识别、DD 虚拟键盘等能力，并支持标准版与 DD 版两种打包模式。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 可视化编辑器 | 基于 CustomTkinter 的节点式编辑器，支持拖拽、连线、缩放、框选 |
| 行为树引擎 | 独立线程执行，支持启动/暂停/停止/恢复 |
| 21 种内置节点 | 5 种复合节点 + 6 种条件节点 + 10 种动作节点 |
| 黑板系统 | 观察者模式的数据共享机制，节点间解耦通信 |
| 序列化 | JSON/YAML/TXT 多格式持久化，版本化数据结构 |
| 撤销/重做 | Command 模式，支持 100 步历史 |
| DD 虚拟键盘 | DD级硬件模拟输入，绕过大多数输入检测 |
| OCR 集成 | 内嵌 RapidOCR + ONNX Runtime，支持中英文识别 |
| 文本处理 | 文本提取与输入功能，支持 OCR 提取、预设文本、文件输入 |
| 脚本录制 | TXT 脚本录制与回放 |
| 双版打包 | 标准版 (PyAutoGUI) 与DD版 (DD64.dll) |
| 放大镜工具 | 区域/位置选择时显示放大镜，精确定位 |
| 随机范围 | 时长参数支持随机变化，模拟人类操作 |
| 坐标偏移 | 条件节点支持坐标偏移，可视化测量工具 |
| 窗口绑定 | 支持绑定目标窗口实现后台运行，自动坐标转换与智能窗口切换 |

### 1.3 技术栈

```
Python 3.10+
├── GUI 框架:    CustomTkinter + Tkinter Canvas
├── 图像处理:    Pillow, OpenCV, imagehash
├── OCR 引擎:    RapidOCR + ONNX Runtime（支持中英文识别）
├── 输入模拟:    PyAutoGUI (标准版) / DD64.dll (DD版)
├── 音频播放:    pygame.mixer
├── 打包工具:    PyInstaller
└── CI/CD:       GitHub Actions
```

## 3 核心组件设计

### 3.1 节点模型 (bt_core/nodes.py)

节点模型是整个系统的核心抽象，采用经典的 Composite 模式设计：

```
Node (ABC)
├── CompositeNode (ABC)
│   ├── SequenceNode     — 顺序执行，全部成功才成功（支持 continue_on_failure、子节点间隔）
│   ├── SelectorNode     — 选择执行，任一成功即成功（支持子节点间隔）
│   ├── ParallelNode     — 并行执行，策略决定成功条件
│   ├── RandomNode       — 随机执行子节点，支持成功策略和完全随机模式
│   └── StartNode        — 根节点，顺序执行子节点，失败后继续执行后续子节点
├── ConditionNode (ABC)
│   ├── _children_running: bool  — 子节点执行状态追踪标志
│   ├── offset: Tuple[int, int]  — 坐标偏移量（相对偏移）
│   ├── _validate_region(region) — 验证区域配置是否有效
│   ├── _get_region_image(context) — 获取区域图像（截图+裁剪）
│   ├── _log_condition_result(success, reason, extra_info) — 记录条件检测结果日志
│   ├── OCRConditionNode     — 文字检测
│   ├── ImageConditionNode   — 图像模板匹配
│   ├── ColorConditionNode   — 颜色检测
│   ├── NumberConditionNode  — 数字识别比较
│   ├── VariableConditionNode— 变量值判断
│   └── TextExtractNode      — 文本提取（OCR识别后提取文本到黑板）
└── ActionNode (ABC)
    ├── _children_running: bool  — 子节点执行状态追踪标志
    ├── KeyPressNode     — 键盘按键
    ├── MouseClickNode   — 鼠标点击
    ├── MouseMoveNode    — 鼠标移动（支持平滑移动）
    ├── MouseScrollNode  — 鼠标滚动
    ├── DelayNode        — 延时等待
    ├── SetVariableNode  — 设置变量
    ├── ScriptNode       — 执行脚本
    ├── CodeNode         — 执行代码
    ├── AlarmNode        — 播放报警
    └── TextInputNode    — 文本输入（支持提取值/预设/文件三种输入模式）
```

**节点核心接口：**

| 方法 | 说明 |
|------|------|
| `tick(context)` | 执行节点逻辑，返回 NodeStatus |
| `reset(reset_counters)` | 重置节点状态，`reset_counters` 控制是否重置计数器 |
| `abort(context)` | 中止节点执行 |
| `add_child(child)` | 添加子节点 |
| `remove_child(child)` | 移除子节点 |
| `to_dict()` | 序列化为字典 |
| `from_dict(data)` | 从字典反序列化 |
| `_execute_children(context)` | 执行子节点（通用实现，位于 Node 基类） |

**子节点执行逻辑统一：**

`_execute_children()` 方法已统一到 `Node` 基类，`ConditionNode` 和 `ActionNode` 不再各自实现：

```python
class Node:
    def __init__(self, ...):
        ...
        self._child_index = 0
        self._children_running = False
    
    def _execute_children(self, context) -> NodeStatus:
        """执行子节点（通用实现）"""
        while self._child_index < len(self.children):
            child = self.children[self._child_index]
            status = child.tick(context)
            if status == NodeStatus.RUNNING:
                self._children_running = True
                return NodeStatus.RUNNING
            if status != NodeStatus.SUCCESS:
                self._child_index = 0
                self._children_running = False
                return status
            self._child_index += 1
        self._child_index = 0
        self._children_running = False
        return NodeStatus.SUCCESS
``` |

**节点保护机制：**

节点支持保护属性 `_is_protected`，防止被误删除：

| 节点类型 | 默认保护状态 | 说明 |
|----------|-------------|------|
| StartNode | 受保护 | 根节点不可删除，确保行为树结构完整 |
| 其他节点 | 不受保护 | 可自由删除 |

```python
# 检查节点是否受保护
if not node._is_protected:
    # 允许删除操作
    pass
```

**节点日志输出机制：**

所有节点在执行成功或失败时都会通过 LogManager 输出运行日志：

| 日志类型 | 触发时机 | 输出内容 |
|----------|----------|----------|
| 成功日志 | 节点返回 SUCCESS | 节点类型、节点名称 |
| 失败日志 | 节点返回 FAILURE | 节点类型、节点名称、失败原因 |
| 中止日志 | 节点被 abort() | 节点类型、节点名称 |

**失败原因输出规范：**

| 节点类型 | 失败原因示例 |
|----------|-------------|
| SequenceNode | "子节点 [节点名] 执行失败" |
| SelectorNode | "所有子节点执行失败" |
| ParallelNode | "成功 2/5 个子节点" |
| RandomNode | "成功 2/5 个子节点" 或 "所有子节点都执行失败" |
| StartNode | "部分子节点执行失败" |
| OCRConditionNode | "未找到关键词: '登录'，识别到的文本: 注册 取消" |
| ImageConditionNode | "未找到匹配的图像，最高置信度: 45.23%，阈值: 80.00%" |
| ColorConditionNode | "未找到匹配的颜色 #FF0000，容差: 30" |
| NumberConditionNode | "未识别到数字，识别到的文本: ABC DEF" |
| VariableConditionNode | "变量 'count' 不存在" 或 "变量 'count' 值为 5，不满足 > 10" |
| TextExtractNode | "未识别到文本" 或 "未提取到匹配的文本" |
| TextInputNode | "未获取到文本内容" 或 "文件不存在: {path}" |

**成功日志简化原则：**

成功日志仅输出节点类型和名称，不包含额外信息，保持简洁：

```python
LogManager.instance().log_success(
    node_type="鼠标移动节点",
    node_name="移动到按钮位置"
)
```

**节点状态管理机制：**

节点在重试（retry）和重复执行（repeat）时需要正确管理状态：

| 方法 | 说明 |
|------|------|
| `_reset_for_retry()` | 重试时重置状态，设置 `status = RUNNING`，保留重试计数器 |
| `_reset_for_repeat()` | 重复执行时重置状态，设置 `status = RUNNING`，保留重复计数器 |
| `_execute_with_decorators()` | 装饰器模式处理 retry/repeat/timeout |

**重要：** `_reset_for_retry()` 和 `_reset_for_repeat()` 忺须将 `status` 设置为 `RUNNING`，而非 `SUCCESS`。这是因为在重试/重复期间，节点仍处于执行状态，需要返回 `RUNNING` 让引擎继续 tick。

**`_reset_for_retry()` 必须重置的字段（V1.3.2b 修复）：**

| 字段 | 说明 | 重置值 |
|------|------|--------|
| `status` | 节点状态 | `RUNNING` |
| `_tick_count` | tick 计数 | `0` |
| `_start_time` | 开始时间 | `None` |
| `_child_index` | 子节点执行索引（基类） | `0` |
| `_children_running` | 子节点运行标志 | `False` |
| `current_index` | 组合节点子节点索引（CompositeNode） | `0` |
| `_last_child_finish_time` | 子节点完成时间（SequenceNode） | `None` |
| `_last_child_time` | 子节点时间（SelectorNode） | `0` |
| 所有子节点 | 递归调用 `child.reset()` | — |

**修复背景：** 在 V1.3.2b 之前，`_reset_for_retry()` 未重置 `current_index`、`_child_index`、`_children_running` 等字段，导致组合节点重试时状态不一致，可能出现：
- ConditionNode 跳过条件检查直接执行子节点
- SequenceNode 从错误的子节点索引继续执行
- 状态灯显示与实际执行结果不一致

**子节点执行状态追踪机制：**

ActionNode 和 ConditionNode 在执行子节点时需要追踪子节点的执行状态，避免在子节点运行期间重复执行动作/条件检测：

| 属性 | 说明 |
|------|------|
| `_children_running` | 布尔标志，标记子节点是否正在执行 |

**状态追踪流程：**

```
ActionNode/ConditionNode 执行流程:
┌─────────────────────────────────────────────────────────────────────┐
│  1. 首次 tick:                                                       │
│     - 执行动作/条件检测                                               │
│     - 若成功且有子节点:                                               │
│       - 通知 GUI 节点状态为 SUCCESS                                   │
│       - 设置 _children_running = True                                │
│       - 开始执行子节点                                                │
│                                                                      │
│  2. 后续 tick:                                                       │
│     - 检查 _children_running 标志                                    │
│     - 若为 True，跳过动作/条件检测，继续执行子节点                      │
│     - 子节点执行完成后，重置 _children_running = False                │
└─────────────────────────────────────────────────────────────────────┘
```

**GUI 状态通知机制：**

当 ActionNode 或 ConditionNode 成功执行后需要执行子节点时，必须先通知 GUI 节点状态为 SUCCESS，确保界面正确显示节点状态：

```python
# 在执行子节点前通知 SUCCESS 状态
if status == NodeStatus.SUCCESS and self.children:
    context.notify_node_status(self.node_id, "success")
    self._children_running = True
    return self._execute_children(context)
```

**reset 方法签名：**

所有节点的 `reset()` 方法应接受 `reset_counters: bool = True` 参数：

```python
def reset(self, reset_counters: bool = True) -> None:
    """
    重置节点状态
    
    Args:
        reset_counters: 是否重置重试/重复计数器，默认 True
    """
    if reset_counters:
        self._retry_count = 0
        self._repeat_count = 0
    self.status = NodeStatus.SUCCESS
    self._tick_count = 0
    self._start_time = None
```

### 3.2 执行引擎 (bt_core/engine.py)

BehaviorTreeEngine 负责行为树的生命周期管理：

- **执行模式**：独立守护线程循环 tick
- **Tick 间隔**：默认 10ms，可配置
- **状态回调**：`on_status_change` / `on_node_status` 通知外部
- **线程安全**：`threading.Lock` 保护启动/停止操作
- **可中断等待**：使用 `threading.Event` 替代 `time.sleep`，支持即时停止
- **执行统计**：集成统计收集器，Debug 环境自动启用

**可中断等待机制：**

```python
def _run_loop(self):
    while not self._stop_event.is_set():
        if not self._pause_event.wait(0.1):
            continue
        self._tick()
        self._stop_event.wait(self._tick_interval)
```

### 5.1 输入控制器

class BaseInputController(ABC):

**DDVirtualInput 关键流程：**

### 5.2 OCR 管理器

OCRManager 封装 RapidOCR + ONNX Runtime，单例模式：

- `recognize(image, keywords, language, preprocess_mode, region)` — 关键词检测，返回 (found, position, all_text)
- `recognize_number(image, language, preprocess_mode, extract_mode, extract_pattern, min_confidence)` — 数字识别，返回 (found, value, all_text)
- `recognize_number_with_position(...)` — 数字识别（带位置），返回 (found, value, all_text, position)
- `get_all_text(image, language, preprocess_mode)` — 全文识别
- `clear_cache()` — 清空结果缓存

**结果缓存机制：**

为提升性能，OCRManager 内置结果缓存：

| 配置 | 值 | 说明 |
|------|-----|------|
| DEFAULT_CACHE_TTL | 1.0 秒 | 缓存过期时间 |
| MAX_CACHE_SIZE | 50 条 | 最大缓存条目数 |

缓存键基于图像哈希和参数哈希生成，相同图像+参数的重复识别直接返回缓存结果。

**图像预处理模式：**

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| normal (默认) | 灰度→对比度增强→锐化→二值化 | 标准文字识别 |
| game (复杂色彩) | 放大→中值滤波→灰度→对比度增强→锐化→二值化 | 游戏界面、小字体 |
| adaptive (自适应) | CLAHE对比度增强→自适应二值化 | 低对比度、渐变背景 |
| auto (自动调优) | 根据图像特征自动选择最佳预处理参数 | 所有场景，推荐使用 |

**数字提取模式：**

| 模式 | 说明 |
|------|------|
| 无规则 | 提取第一个数字 |
| x/y | 提取分数形式的分子（如 "3/5" 提取 3） |
| 自定义 | 使用 `*` 通配符定义提取模式（如 "HP:*" 提取 "HP:100" 中的 100） |

def _get_virtual_screen_bounds(self) -> Tuple[int, int, int, int]:
    """获取虚拟桌面边界（支持多显示器）"""
    try:
        import screeninfo
        monitors = screeninfo.get_monitors()
        if monitors:
            min_x = min(monitor.x for monitor in monitors)
            min_y = min(monitor.y for monitor in monitors)
            max_x = max(monitor.x + monitor.width for monitor in monitors)
            max_y = max(monitor.y + monitor.height for monitor in monitors)
            return (min_x, min_y, max_x, max_y)
    except ImportError:
        pass
    
    # Windows API 备用方案
    try:
        import ctypes
        user32 = ctypes.windll.user32
        virtual_left = user32.GetSystemMetrics(76)   # SM_XVIRTUALSCREEN
        virtual_top = user32.GetSystemMetrics(77)    # SM_YVIRTUALSCREEN
        virtual_width = user32.GetSystemMetrics(78)  # SM_CXVIRTUALSCREEN
        virtual_height = user32.GetSystemMetrics(79) # SM_CYVIRTUALSCREEN
        return (virtual_left, virtual_top, 
                virtual_left + virtual_width, 
                virtual_top + virtual_height)
    except Exception:
        pass
    
    # 最终回退
    return (0, 0, 1920, 1080)
```

### 5.7 随机范围辅助工具 (bt_utils/helpers.py)

提供随机值计算工具函数，用于模拟更自然的人类操作：

**核心函数：**

```python
def get_random_value(base_value: Union[int, float], random_range: Union[int, float] = 0, 
                     min_value: Union[int, float, None] = None) -> Union[int, float]:
    """
    计算带随机范围的值
    
    Args:
        base_value: 基础值（支持整数和浮点数）
        random_range: 随机范围（±）
        min_value: 最小值限制（可选）
    
    Returns:
        计算后的随机值
    """
    if random_range <= 0:
        return base_value
    
    min_val = base_value - random_range
    max_val = base_value + random_range
    
    if min_value is not None:
        min_val = max(min_val, min_value)
    
    if isinstance(base_value, float):
        return random.uniform(min_val, max_val)
    return random.randint(int(min_val), int(max_val))


def get_random_duration(base_ms: int, random_ms: int = 0) -> int:
    """
    计算随机延时时长（毫秒）
    
    Args:
        base_ms: 基础时长（毫秒）
        random_ms: 随机范围（±毫秒）
    
    Returns:
        计算后的延时时长（毫秒）
    """
    return get_random_value(base_ms, random_ms, min_value=0)


def get_random_interval(base_ms: int, random_ms: int = 0) -> float:
    """
    计算随机间隔时长（秒）
    
    Args:
        base_ms: 基础时长（毫秒）
        random_ms: 随机范围（±毫秒）
    
    Returns:
        计算后的间隔时长（秒）
    """
    ms = get_random_value(base_ms, random_ms, min_value=0)
    return ms / 1000.0
```

### 5.8 UI 更新分发器 (bt_utils/ui_dispatcher.py)

UIUpdateDispatcher 是解决 tkinter 线程安全问题的核心组件，采用事件驱动 + 定时轮询混合模式：

**设计背景：**

tkinter 的 GUI 操作必须在主线程执行，但行为树引擎在独立线程中运行。直接从后台线程调用 `update()` 或 `event_generate()` 可能导致界面假死或更新延迟。

**核心机制：**

| 机制 | 说明 |
|------|------|
| 线程安全队列 | `Queue` 存储待处理的 UI 更新任务 |
| 定时轮询 | 30ms 间隔检查队列，确保不遗漏更新 |
| 节点状态缓存 | 合并相同节点的重复状态更新 |
| 后台轮询守护 | 当 `_widget` 为 None 时，后台线程继续尝试调度 |

**数据流：**

```
后台线程 (Engine)                     主线程 (GUI)
       │                                   │
       │  dispatch_node_status()           │
       │  ┌──────────────────────┐         │
       │  │ Queue.put(task)      │         │
       │  └──────────────────────┘         │
       │                                   │
       │  同时：定时轮询检查队列            │
       │  ┌──────────────────────┐         │
       │  │ 每30ms检查队列        │────────▶│
       │  │ 有任务则处理          │         │
       │  └──────────────────────┘         │
       │                                   │  _process_updates()
       │                                   │  ┌──────────────────────┐
       │                                   │  │ 从队列取出任务        │
       │                                   │  │ 执行回调更新UI        │
       │                                   │  └──────────────────────┘
```

**关键方法：**

| 方法 | 说明 |
|------|------|
| `attach(widget)` | 绑定到 GUI 组件 |
| `dispatch_node_status()` | 分发节点状态更新 |
| `dispatch_log_flush()` | 分发日志刷新请求 |
| `start_polling()` | 启动定时轮询 |
| `stop_polling()` | 停止定时轮询 |
| `process_pending()` | 立即处理待处理任务 |

**⚠️ 防止假卡死的重要注意事项：**

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 启动时假卡死 | 轮询未在编辑器初始化时启动 | 在 `_init_ui_dispatcher()` 中立即调用 `start_polling()` |
| 手动停止后再次运行异常 | 停止时调用 `stop_polling()` 导致轮询停止 | **不要在停止时调用 `stop_polling()`**，让轮询持续运行 |
| 轮询永久停止 | `_widget` 为 None 时轮询直接返回 | 使用后台线程守护，当 `_widget` 可用时切换回主线程轮询 |
| `after(0, ...)` 不可靠 | 从后台线程调用可能不生效 | 完全依赖定时轮询机制，不使用 `after(0, ...)` |

**正确的初始化流程：**

```python
def _init_ui_dispatcher(self):
    from bt_utils.ui_dispatcher import UIUpdateDispatcher
    self._dispatcher = UIUpdateDispatcher.get_instance()
    self._dispatcher.attach(self)
    self._dispatcher.start_polling()  # 必须在初始化时启动，而不是运行时

def _start_ui_polling(self):
    pass  # 已在初始化时启动，此处为空

def _stop_ui_polling(self):
    pass  # 不要停止轮询，让它持续运行
```

**轮询守护机制：**

```python
def _poll(self):
    if not self._polling_active:
        return
    
    widget = self._widget
    
    try:
        if widget is not None:
            pending = self._task_queue.qsize()
            if pending > 0:
                self._process_updates()
    except Exception:
        pass
    
    if self._polling_active:
        if self._widget is not None:
            self._widget.after(self._polling_interval_ms, self._poll)
        else:
            # 后台线程守护，等待 _widget 可用
            threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()

def _poll_daemon(self):
    """后台线程轮询，用于在 _widget 为 None 时继续尝试"""
    if self._polling_active and self._widget is not None:
        self._widget.after(0, self._poll)  # 切换回主线程轮询
    elif self._polling_active:
        threading.Timer(self._polling_interval_ms / 1000, self._poll_daemon).start()
```

## 8 安全机制

### 8.1 Zip Slip 漏洞防护

`PackageImporter` 在导入 ZIP 项目时检测路径遍历攻击：

```python
def validate_package(self, zip_path):
    with zipfile.ZipFile(zip_path, 'r') as zipf:
        for name in zipf.namelist():
            normalized = os.path.normpath(name)
            if normalized.startswith('..') or os.path.isabs(normalized):
                return False, "ZIP 包含路径遍历条目，可能是恶意文件"
    return True, ""
```

### 8.2 路径遍历防护

`PathResolver` 在解析相对路径时检测路径逃逸：

```python
def to_absolute(self, relative_path: str) -> str:
    abs_path = os.path.normpath(os.path.join(self.project_root, relative_path))
    normalized_root = os.path.normpath(self.project_root)
    if not (abs_path.startswith(normalized_root + os.sep) or abs_path == normalized_root):
        raise ValueError(f"路径逃逸: '{relative_path}' 解析后超出项目根目录")
    return abs_path
```

### 8.3 CodeNode 安全沙箱

`CodeNode` 在执行外部代码前进行 AST 安全检查：

```python
class CodeSecurityChecker:
    ALLOWED_NODES = {

    }
    FORBIDDEN_NAMES = {
        'import', '__import__', 'eval', 'exec', 'open',
        'os', 'sys', 'subprocess', 'socket', ...
    }

`SettingsManager` 使用 `copy.deepcopy` 避免默认值污染：

    screen_region[1] - window_rect[1],

```

### 10.7 窗口切换例外机制

**SKIP_WINDOW_SWITCH 属性：**

某些节点不需要操作窗口，可以设置 `SKIP_WINDOW_SWITCH = True` 来跳过窗口切换逻辑：

| 节点类型 | SKIP_WINDOW_SWITCH | 说明 |
|---------|-------------------|------|
| DelayNode | True | 延时节点，仅等待时间 |
| SetVariableNode | True | 设变量节点，操作黑板数据 |
| AlarmNode | True | 报警节点，播放声音 |
| CodeNode | True | 代码节点，执行外部代码 |
| ScriptNode | True | 脚本节点，执行脚本录制回放 |
| MouseClickNode | False | 需要窗口操作 |
| MouseMoveNode | False | 需要窗口操作 |
| KeyPressNode | False | 需要窗口操作 |
| 其他动作节点 | False | 默认需要窗口操作 |

**实现原理：**

```python
class ActionNode(Node):
    SKIP_WINDOW_SWITCH = False  # 基类默认不跳过

    def _tick_internal(self, context):
        bound_window = context.get_bound_window()
        
        # 只有绑定了窗口且节点不跳过窗口切换时才执行切换逻辑
        if bound_window and not self.SKIP_WINDOW_SWITCH:
            # 窗口切换逻辑...
        else:
            # 直接执行动作
            status = self._execute_action(context)
```

输入流程:

    if char == '\n':  → 按下 Enter
    elif char == '\t': → 按下 Tab
    else:
      pyperclip.copy(char)    → 复制字符到剪贴板
      Ctrl+Down               → 按下 Ctrl
      V press                 → 粘贴
      Ctrl+Up                 → 抬起 Ctrl
    sleep(input_delay)        → 等待输入间隔
```

**预设文本执行模式：**

| 模式 | 中文显示 | 说明 |
|------|---------|------|
| sequential | 顺序 | 按顺序循环使用预设文本 |
| random | 随机 | 随机选择预设文本 |

### 11.6 资源管理

TextInputNode 的文件模式使用 `file_path` 属性，该属性已注册到 ResourceService：

```python
# ResourceService 中的注册
RESOURCE_KEYS = ['template_path', 'script_path', 'code_path', 'sound_path', 'file_path']
RESOURCE_TYPE_MAP = {'file_path': 'data'}  # 文件保存到 data/config/ 目录
```

这确保了文件模式下的文本文件会被正确导入到项目文件夹中，并在项目导出/导入时保持路径一致。
